# 履歴カード入力方法ラベル削除テスト設計

## 目的
- ホーム画面と履歴一覧で共通利用している履歴カードから、右上の入力方法ラベルが削除され続けることをCIで担保する。
- タイトル切り替えや日付表示など既存の主要情報を維持したまま、不要な補助情報のみが除去されていることを確認する。

## 対象
- `apps/web/src/components/app/history/HistoryCard.tsx`
- `apps/web/src/components/app/screens/HomeTabScreen.tsx`
- `apps/web/src/components/app/screens/HistoriesTabScreen.tsx`
- `apps/web/scripts/check-home-history-card-presentation-spec.mjs`
- `apps/web/scripts/check-history-list-card-presentation-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### HIC-01 共通履歴カードが入力方法ラベルを描画しない
- 観点: カード右上の補助情報が完全に取り除かれていること
- 確認項目:
  - `HISTORY_STRINGS.labels.inputType[item.inputType]` を参照しない
  - `contextLabel` を定義しない
  - 右上ラベル用の `justify-between` レイアウトを持たない

### HIC-02 ホーム画面の共通履歴カード利用を維持する
- 観点: 表示簡素化後もホーム画面が共通コンポーネント経由で描画されること
- 確認項目:
  - `HomeTabScreen.tsx` が `SharedHistoryCard` を import する
  - `from="home"` を指定して描画する

### HIC-03 履歴一覧の共通履歴カード利用を維持する
- 観点: 履歴一覧側も同一コンポーネントを継続利用し、表示差異が発生しないこと
- 確認項目:
  - `HistoriesTabScreen.tsx` が `SharedHistoryCard` を import する
  - `from="histories"` を指定して描画する

## テスト実装
- `pnpm --filter web test:home-history-card-presentation`
- `pnpm --filter web test:history-list-card-presentation`

## CI適用
- `.github/workflows/ci.yml` で以下ジョブを継続実行する。
  - `Home history card presentation spec test`
  - `History list card presentation spec test`

## PRサマリー記載用テストケース
- HIC-01 共通履歴カードの入力方法ラベル非表示
- HIC-02 ホーム画面での共通履歴カード利用維持
- HIC-03 履歴一覧での共通履歴カード利用維持
