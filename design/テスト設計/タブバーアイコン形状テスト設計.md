# タブバーアイコン形状テスト設計

## 1. 目的

* ホームタブのアイコンが、家の輪郭線と中央ドアを塗りつぶしなしの線画で表現していることを継続検証する。
* テンプレートタブのアイコンが、左上と右下に配置した2つの四角の重なりとして表現され、重なり部分で下の四角が露出しないことを担保する。
* アイコン形状の退行を Node.js スクリプトと CI に組み込み、SVG 差し替え時の視覚仕様崩れを防止する。

## 2. 対象

* `apps/web/src/components/app/navigation/TabBarIcon.tsx`
* `apps/web/scripts/check-tab-bar-icon-shapes-spec.mjs`
* `apps/web/package.json`
* `.github/workflows/ci.yml`

## 3. テスト観点

### 3.1 ホームタブアイコン

* 屋根線、家本体、中央ドアがそれぞれ独立した線画 path として定義されること
* ドアが塗りつぶし矩形ではなく、家本体と調和した輪郭線で描画されること

### 3.2 テンプレートタブアイコン

* 左上の四角が `rect` で定義されること
* 右下の四角が開いた path で定義され、重なり部分では下側の輪郭が見えない構成になっていること
* 旧来の「紙＋罫線」表現が残存していないこと

### 3.3 CI組み込み

* `pnpm --filter web test:tab-bar-icon-shapes` が `apps/web/package.json` に定義されていること
* GitHub Actions の CI で当該テストが実行されること

## 4. テストケース

| ケースID | 観点 | 条件 | 期待結果 |
| --- | --- | --- | --- |
| TBIS-01 | ホームタブアイコン | `TabBarIcon.tsx` を確認する | 屋根・家本体・中央ドアが線画 path で定義され、塗りつぶし矩形が使われていない |
| TBIS-02 | テンプレートタブアイコン | `TabBarIcon.tsx` を確認する | 左上の四角と右下の四角が重なった copy ライクな線画として定義され、旧罫線表現が除去されている |
| TBIS-03 | CI組み込み | `package.json` と `ci.yml` を確認する | 形状検証スクリプトが npm script と CI に登録されている |

## 5. 実装方針

* Node.js スクリプト `apps/web/scripts/check-tab-bar-icon-shapes-spec.mjs` で、ホーム/テンプレートアイコンの SVG 定義と旧表現の除去を静的検証する。
* PR・push 時の CI で継続実行し、アイコン形状の変更が意図したデザインから逸脱していないかを自動検知する。
