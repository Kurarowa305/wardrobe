# 服追加画面 画像ラベル任意表記テスト設計

## 目的

- 服追加画面の画像ラベルが要望どおり「画像ファイル（任意）」へ変更されていることをCIで継続検証する。
- 服編集画面など別画面の同名ラベルへ意図せず影響しないことを静的検査で担保する。

## 対象スクリプト

- `apps/web/scripts/check-clothing-create-image-optional-label-spec.mjs`
- 実行コマンド: `pnpm --filter web test:clothing-create-image-optional-label`

## テストケース

### COL-01 服追加画面の画像ラベルが任意表記になっている
- 観点: 画面仕様の「画像（任意）」と文言変更要望の反映
- 期待結果:
  - `src/features/clothing/strings.ts` の `CLOTHING_STRINGS.create.labels.imageFile` が `画像ファイル（任意）` になっている

### COL-02 服編集画面の画像ラベルは既存文言を維持する
- 観点: 影響範囲の局所化
- 期待結果:
  - `src/features/clothing/strings.ts` の `CLOTHING_STRINGS.edit.labels.imageFile` は `画像ファイル` のままである

## CI適用

- `.github/workflows/ci.yml` に `Clothing create image optional label spec test` を追加し、push時に自動検証する
