# ホーム直近1週間履歴見出しアイコン表示テスト設計

## 1. 目的

* ホーム画面の「直近1週間の履歴」見出し左側に、履歴タブと同一のアイコンが表示されることを静的検証で担保する。
* 見出しアイコンの線色が黒（`#000000`）で描画されること、および本検証が CI に組み込まれていることを保証する。

## 2. 対象

* `apps/web/src/components/app/screens/HomeTabScreen.tsx`
* `apps/web/src/components/ui/tab-bar-icon.tsx`
* `apps/web/scripts/check-home-recent-history-heading-icon-spec.mjs`
* `apps/web/package.json`
* `.github/workflows/ci.yml`

## 3. 検証観点

### 3.1 アイコン再利用

* ホーム画面が `TabBarIcon` を import していること
* 「直近1週間の履歴」見出しに `histories` アイコンを利用していること

### 3.2 レイアウト

* 見出しがアイコンと文言の横並びになっていること
* 見出し文言が `HOME_STRINGS.sections.recentWeekHistories` を継続利用していること

### 3.3 線色制御

* `TabBarIcon` が任意の線色上書きを受け付けること
* ホーム画面から黒（`#000000`）を指定していること

### 3.4 CI 組み込み

* `pnpm --filter web test:home-recent-history-heading-icon` が `apps/web/package.json` に定義されていること
* GitHub Actions の CI で同スクリプトが実行されること

## 4. テストケース

| ID | 観点 | 手順 | 期待結果 |
| --- | --- | --- | --- |
| HRHI-01 | アイコン再利用 | `HomeTabScreen.tsx` を確認する | `TabBarIcon` が import され、`histories` アイコンが見出しに使われている |
| HRHI-02 | 横並びレイアウト | `HomeTabScreen.tsx` を確認する | 見出しがアイコンと文言の横並び構造になっている |
| HRHI-03 | 黒線表示 | `HomeTabScreen.tsx` と `tab-bar-icon.tsx` を確認する | `strokeColor` を受け取り、ホーム画面から `#000000` が指定されている |
| HRHI-04 | CI 組み込み | `package.json` と `ci.yml` を確認する | 検証スクリプトが npm script と CI の両方に登録されている |

## 5. 実施方法

* Node.js スクリプト `apps/web/scripts/check-home-recent-history-heading-icon-spec.mjs` で構造・色指定・CI 組み込みを検証する。
* 既存のホーム画面検証とは責務を分離し、見出しアイコン表示仕様の退行を個別に検知する。
* PR・push 時の CI で継続実行し、ホーム画面の履歴見出しアイコン表示仕様を保護する。
