# アクセントカラー統一テスト設計

## 1. 目的

- 指定された主要導線のアクセントカラーが `#687A88` に統一されることを継続的に検証する
- 対象ボタン内の文字色が白で表示されることを静的テストで担保する
- `＋ 記録する`、`＋ テンプレートを追加`、`＋ 服を追加` の文字を太字かつ大きめに保つ
- タブバーのアクティブ状態も同じアクセントカラー・白文字で表示されることをCIで保証する

## 2. 対象

- `apps/web/src/components/ui/button.tsx`
- `apps/web/src/app/globals.css`
- 以下の default Button を利用する主要導線
  - `＋ 記録する`
  - `＋ テンプレートを追加`
  - `＋ 服を追加`
  - `ワードローブの作成`
  - 記録
  - 保存
  - 追加

## 3. テスト観点

### AC-01 アクセントカラー変数が更新されている

- 観点: グローバルCSSのアクセントカラーが `#687A88` になっていること
- 観点: タブバーのアクティブ状態が白文字で描画されること

### AC-02 primary Button が白文字・指定アクセントカラーになる

- 観点: Button 基盤の default variant が `#687A88` を利用すること
- 観点: ボタン内テキスト色が白であること

### AC-03 指定導線が Button 基盤の default variant を使い続ける

- 観点: ホーム・テンプレート一覧・服一覧の追加導線が default Button のままであること
- 観点: 上記3ボタンの文字色が白、サイズが `text-base`、太さが `font-bold` であること
- 観点: ワードローブ作成、記録、追加、保存の送信導線が default Button のままであること

## 4. テスト手段

- `apps/web/scripts/check-accent-color-spec.mjs` を追加し、対象ファイルの静的検査を行う
- `apps/web/package.json` に `test:accent-color` を追加する
- `.github/workflows/ci.yml` に `Accent color spec test` を追加し、push 時に自動実行する

## 5. CI 組み込み

- 実行コマンド: `pnpm --filter web test:accent-color`
- 期待結果: すべての観点が `✅` で終了し、終了コード `0` を返す
