# アクセントカラー統一テスト設計

## 目的

- 指定アクセントカラー `#687A88` が主要CTAボタンとタブバーのアクティブ状態に一貫して適用されることを静的検証で担保する。
- 共通 `Button` の default variant と `globals.css` のテーマ変数を基点に色を統一する実装方針が維持されることを確認する。
- 退行防止のため、テストスクリプトを CI に組み込んで push 時に自動実行する。

## 対象

- `apps/web/src/components/ui/button.tsx`
- `apps/web/src/app/globals.css`
- `apps/web/src/components/app/screens/WardrobeCreateScreen.tsx`
- `apps/web/src/components/app/screens/HomeTabScreen.tsx`
- `apps/web/src/components/app/screens/TemplatesTabScreen.tsx`
- `apps/web/src/components/app/screens/ClothingsTabScreen.tsx`
- `apps/web/src/components/app/screens/TemplateForm.tsx`
- `apps/web/src/components/app/screens/ClothingCreateScreen.tsx`
- `apps/web/src/components/app/screens/ClothingEditScreen.tsx`
- `apps/web/src/components/app/screens/RecordByTemplateScreen.tsx`
- `apps/web/src/components/app/screens/RecordByCombinationScreen.tsx`
- `apps/web/src/components/app/navigation/TabBar.tsx`
- `apps/web/scripts/check-accent-color-spec.mjs`
- `apps/web/package.json`
- `.github/workflows/ci.yml`

## テスト観点

### 1. 共通カラー定義

- `globals.css` の `--primary` が `#687A88` であること
- `globals.css` の `--primary-soft` がタブのアクティブ背景用に設定されていること
- `button.tsx` の default variant が `#687A88` 系の配色を参照すること

### 2. 対象CTAへの適用

- ワードローブ作成ボタンが default ボタンで描画されること
- ホームの「＋ 記録する」ボタンが default ボタンで描画されること
- テンプレート一覧の「＋ テンプレートを追加」ボタンが default ボタンで描画されること
- 服一覧の「＋ 服を追加」ボタンが default ボタンで描画されること
- テンプレート追加/編集、服追加/編集、記録画面の送信ボタンが default ボタンで描画されること

### 3. タブバー適用

- タブバーのアクティブ状態が `var(--primary)` と `var(--primary-soft)` を利用すること
- `TabBar.tsx` 側で active 状態に `is-active` クラスを付与していること

### 4. CI組み込み

- `pnpm --filter web test:accent-color` が `apps/web/package.json` に定義されていること
- GitHub Actions の `ci.yml` で `Accent color spec test` が実行されること

## テストケース

| ケースID | 観点 | 条件 | 期待結果 |
| --- | --- | --- | --- |
| ACS-01 | 共通カラー定義 | `globals.css` を確認する | `--primary` が `#687A88`、`--primary-soft` がアクティブ背景色として設定されている |
| ACS-02 | 共通ボタン配色 | `button.tsx` を確認する | default variant が `#687A88` を使う |
| ACS-03 | タブバー適用 | `globals.css` と `TabBar.tsx` を確認する | アクティブタブが primary 系変数と `is-active` クラスで描画される |
| ACS-04 | 対象CTA適用 | 各 Screen ファイルを確認する | 指定対象CTAが default ボタンとして実装されている |
| ACS-05 | CI組み込み | `package.json` と `ci.yml` を確認する | accent color テストが npm script と CI に追加されている |

## 実装方針

- Node.js スクリプト `apps/web/scripts/check-accent-color-spec.mjs` で対象ファイルの必要文字列を検証する。
- 色定義は共通部品に集約し、個別画面では default variant の利用有無を確認することで、指定箇所の統一を担保する。
