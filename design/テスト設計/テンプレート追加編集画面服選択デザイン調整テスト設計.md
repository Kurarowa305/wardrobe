# テンプレート追加編集画面 服選択デザイン調整テスト設計

## 目的

- テンプレート追加・編集画面の服選択UIが、記録画面の服組み合わせ選択と同系統の見た目・操作導線になっていることをCIで継続検証する
- 服選択セクションから旧来の外側カード表現が除去されていることを担保する
- 各服カードにアイコンと右端のチェック領域があり、選択時にアクセントカラーで状態が視認できることを担保する
- 追加/保存ボタンが画面下部固定表示になり、コンテンツがボタンに隠れない余白が確保されていることを担保する

## 対象スクリプト

- `apps/web/scripts/check-template-form-selection-layout-spec.mjs`
- 実行コマンド: `pnpm --filter web test:template-form-selection-layout`

## テストケース

### TFL-01 服選択セクションは外側カードを持たないプレーンレイアウトになっている
- 観点: ユーザー要望の「外側のカード表示削除」が反映されているか
- 期待結果:
  - `TemplateForm.tsx` の服選択 `fieldset` が `border-0 p-0` のプレーンレイアウトで描画される
  - `legend` は余白を抑えたテキスト表示になる
  - 旧来の `rounded-md border border-slate-200 p-3` が残っていない

### TFL-02 各服カードがアイコン付きの選択行になっている
- 観点: 記録画面の服選択と同じ系統の情報密度・視認性になっているか
- 期待結果:
  - `TemplateForm.tsx` が `TabBarIcon` を import している
  - 各カードが `grid-cols-[28px_minmax(0,1fr)_40px]` の3カラム構成で描画される
  - 左側に `clothings` アイコンが表示される
  - ネイティブ checkbox は `sr-only` にしてカード全体で選択できる

### TFL-03 右端に独立したチェック領域があり、選択時はアクセントカラーになる
- 観点: 選択状態が視覚的に明快か
- 期待結果:
  - 右側に `flex justify-end` のチェック領域がある
  - 選択時は `border-[var(--primary)] bg-[var(--primary)] text-white` で表示される
  - 未選択時は `border-slate-300 bg-white text-transparent` で非選択状態を表現する
  - チェック記号 `✓` が表示される

### TFL-04 追加/保存ボタンが画面下部固定表示になっている
- 観点: 保存導線が常に視界に入り、長いリストでも操作しやすいか
- 期待結果:
  - `TemplateForm.tsx` のフォームが `pb-24` を持ち、固定ボタンぶんの余白を確保している
  - 送信ボタンは `fixed bottom-0 ...` の固定フッター内に描画される
  - 固定フッター内のボタンは横幅いっぱいで表示される

## CI適用

- `.github/workflows/ci.yml` に `Template form selection layout spec test` を追加し、PR時に自動検証する
