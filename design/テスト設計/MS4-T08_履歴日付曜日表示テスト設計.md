# MS4-T08 履歴日付曜日表示テスト設計

## 目的
- 履歴詳細画面・履歴一覧カード・ホームの履歴カードで、履歴日付が曜日付き `YYYY/MM/DD (曜)` 形式で統一表示されることをCIで担保する。

## 対象
- `apps/web/src/features/history/date.ts`
- `apps/web/src/components/app/history/HistoryCard.tsx`
- `apps/web/src/components/app/screens/HistoryDetailScreen.tsx`
- `apps/web/scripts/check-history-date-weekday-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点
1. 履歴日付フォーマッタが `yyyymmdd` を曜日付き `YYYY/MM/DD (曜)` に整形する。
2. 履歴一覧カード/ホーム共通カードが共通フォーマッタを利用して曜日付き日付を表示する。
3. 履歴詳細画面が同じ共通フォーマッタを利用して曜日付き日付を表示する。
4. 専用テストスクリプトが package script と CI に登録され、PR時に自動実行される。

## テストケース
- **HWD-01**: `formatHistoryDate` が曜日配列と `Date#getDay()` を利用して `YYYY/MM/DD (曜)` を返す。
- **HWD-02**: `HistoryCard.tsx` が `formatHistoryDate(item.date)` を利用する。
- **HWD-03**: `HistoryDetailScreen.tsx` が `formatHistoryDate(historyQuery.data.date)` を利用する。
- **HWD-04**: `apps/web/package.json` と `.github/workflows/ci.yml` に `test:history-date-weekday` が登録される。

## 実行コマンド
- `pnpm --filter web test:history-date-weekday`
