# MS5-T02 テンプレート記録画面テスト設計

## 目的

- MS5-T02（記録：テンプレで記録）の完了条件を継続的に検証する
- テンプレート選択から履歴作成までの導線・入力・遷移をCIで担保する

## 対象範囲

- `apps/web/src/components/app/screens/RecordByTemplateScreen.tsx`
- `apps/web/src/features/record/strings.ts`
- `apps/web/scripts/check-record-template-screen-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### RTS-01 テンプレート記録画面が所定ファイルに存在する
- 観点: 記録フローのテンプレート記録画面が設計どおり専用 Screen として存在すること

### RTS-02 AppLayout でタイトルと戻り先が記録方法選択に設定される
- 観点: スタック画面としてタイトルと戻る導線が正しく構成されていること
- 期待値:
  - `RECORD_STRINGS.byTemplate.title` を画面タイトルに利用する
  - `ROUTES.recordMethod(wardrobeId)` を戻り先に利用する

### RTS-03 日付入力欄が date input と必須エラーメッセージ付きで実装される
- 観点: 履歴作成時に日付が必須入力として扱われること
- 期待値:
  - `type="date"` の入力欄を持つ
  - 空入力時に `dateRequired` 文言を表示できる

### RTS-04 テンプレート一覧取得に template query hook を用い、単一選択 UI を描画する
- 観点: テンプレート一覧を取得して単一選択 UI を提供できること
- 期待値:
  - `useTemplateList(wardrobeId, ...)` を利用する
  - 各テンプレートを radio で選択できる

### RTS-05 選択中テンプレートを確認できる領域を持つ
- 観点: 記録前に、選択したテンプレートを見返せること
- 期待値:
  - `selectedTemplate` を基準に選択中テンプレートを導出する
  - 選択中表示に `selected` 文言を用いる

### RTS-06 記録ボタンが create history mutation を呼び、API仕様の yyyymmdd に変換した templateId で履歴を作成する
- 観点: テンプレート記録として履歴作成 API に正しい payload を渡せること
- 期待値:
  - `useCreateHistoryMutation(wardrobeId)` を利用する
  - 日付入力値を `yyyymmdd` に正規化してから `mutateAsync` を呼ぶ
  - `templateId: selectedTemplateId` を渡す

### RTS-07 記録成功後にホームへ戻る
- 観点: 記録完了後の導線が画面遷移図どおりホームへ戻ること
- 期待値:
  - 成功時に `router.push(ROUTES.home(wardrobeId))` を実行する

### RTS-08 テンプレート記録向け文言が record strings に定義される
- 観点: 画面固有文言が `record/strings.ts` に集約されていること
- 期待値:
  - タイトル、ラベル、選択中表示、必須/失敗文言が定義されている

## テストスクリプト

- スクリプト: `pnpm --filter web test:record-template-screen`
- 実体: `apps/web/scripts/check-record-template-screen-spec.mjs`

## CI適用方針

- GitHub Actions の `CI` workflow に `Record template screen spec test` を追加する
- 既存の画面仕様テスト群と同列に配置し、push ごとに継続実行する

## PRサマリーに記載するテストケース

- RTS-01 テンプレート記録画面のファイル配置
- RTS-02 AppLayout のタイトル / 戻り導線
- RTS-03 日付入力と必須バリデーション
- RTS-04 テンプレート一覧取得と単一選択 UI
- RTS-05 選択中テンプレートの確認表示
- RTS-06 yyyymmdd 変換を含む履歴作成 mutation の payload
- RTS-07 記録完了後のホーム遷移
- RTS-08 文言定義の配置
