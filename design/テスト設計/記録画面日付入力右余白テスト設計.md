# 記録画面日付入力右余白テスト設計

## 目的
- 記録画面の日付入力欄で、右端のカレンダーアイコン側にも左側と同等の余白が確保されることを継続的に検証する。
- 共通 Input 基盤の調整でテンプレート記録画面・服組み合わせ記録画面の双方へ反映されることを CI で担保する。

## 対象ファイル
- `apps/web/src/components/ui/input.tsx`
- `apps/web/src/app/globals.css`
- `apps/web/src/components/app/screens/RecordByTemplateScreen.tsx`
- `apps/web/src/components/app/screens/RecordByCombinationScreen.tsx`
- `apps/web/scripts/check-record-date-input-spacing-spec.mjs`

## テスト観点
1. Input 基盤が `type="date"` を判定し、専用の `date-input` クラスを付与する。
2. `globals.css` に `::-webkit-calendar-picker-indicator` と `::-webkit-datetime-edit` 向けの右余白スタイルが定義されている。
3. 記録画面 2 画面が共通 Input 基盤の `type="date"` を利用しており、余白調整が両画面へ適用される。

## テストケース
### RDIS-01 Input 基盤が date input を判定して専用クラスを付与する
- 観点: 日付入力欄のみを対象に右余白調整を適用し、他 input の既存スタイルへ影響を広げないこと
- 期待結果:
  - `const isDateInput = type === "date";` が存在する
  - `isDateInput && "date-input px-3 pr-3"` が className 合成に含まれる

### RDIS-02 date input のカレンダーアイコン右余白を globals.css で確保する
- 観点: ネイティブ date input の右端に寄る表示を CSS で補正できること
- 期待結果:
  - `.date-input::-webkit-calendar-picker-indicator` が存在する
  - `margin-right: 12px;` が設定される
  - `.date-input::-webkit-datetime-edit` が存在する
  - `padding-inline-end: 12px;` が設定される

### RDIS-03 記録画面 2 画面が共通 Input 基盤の date input を利用する
- 観点: 共通 Input の調整だけでテンプレート記録画面・服組み合わせ記録画面の双方に反映されること
- 期待結果:
  - `RecordByTemplateScreen` が `<Input type="date" />` を利用している
  - `RecordByCombinationScreen` が `<Input type="date" />` を利用している

## 実行方法
```bash
pnpm --filter web test:record-date-input-spacing
```

## CI 適用方針
- GitHub Actions の `CI` workflow に `pnpm --filter web test:record-date-input-spacing` を追加し、push ごとに継続実行する。
