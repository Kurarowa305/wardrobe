# MS5-T02 記録：テンプレで記録テスト設計

## 目的
- MS5-T02（テンプレ選択→履歴作成）の完了条件を継続的に検証する。
- 記録画面の日付入力が API 契約の `yyyymmdd` に変換されることを担保する。

## 対象
- `apps/web/src/components/app/screens/RecordByTemplateScreen.tsx`
- `apps/web/src/features/record/strings.ts`
- `apps/web/scripts/check-record-by-template-screen-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### RTT-01 画面骨格と導線
- `RecordByTemplateScreen` が client component として実装される。
- `AppLayout` にタイトルと戻る導線が設定される。
- 成功時にホーム画面へ遷移する。

### RTT-02 入力 UI
- 日付入力が `type="date"` で提供される。
- テンプレート選択が単一選択（radio）で提供される。
- 必須入力不足時のエラーメッセージが定義される。

### RTT-03 API 契約との整合
- 送信時 payload に `templateId` を含める。
- 日付入力値 `yyyy-mm-dd` を `yyyymmdd` へ変換して送信する。
- API 契約に合わないフォーマットをそのまま送らない。

### RTT-04 一覧取得と追加読み込み
- テンプレート一覧の読み込み中・失敗・空状態の文言を表示できる。
- `nextCursor` が存在する場合に追加読み込みボタンを表示できる。
- 追加読み込みボタンの文言が strings に定義される。

## テストケース
| ID | 観点 | 検証内容 |
| --- | --- | --- |
| RTT-01 | 画面骨格 | `"use client"`、`AppLayout`、戻る導線が存在する |
| RTT-02 | 入力UI | 日付入力とテンプレ単一選択（radio）が存在する |
| RTT-03 | API 契約 | `formatDateForApi` が `-` を除去し `date` を `yyyymmdd` で送る |
| RTT-04 | 成功導線 | 作成成功後に `ROUTES.home(wardrobeId)` へ遷移する |
| RTT-05 | 追加読み込み | `handleLoadMore` と load more 文言が存在する |
| RTT-06 | 文言管理 | loading / required / submit error などの文言が `record/strings.ts` にある |

## 実行方法
- `pnpm --filter web test:record-by-template-screen`

## CI 適用
- GitHub Actions の `ci.yml` に `Record by template screen spec test` を追加し、PR/push で自動実行する。
