# 服詳細画像表示改善テスト設計

## 目的

- 服詳細画面の画像が固定高トリミングで見切れず、画像比率に追従して全体表示されることを継続的に確認する。
- 画像未設定時の `no image` フォールバックと、既存の詳細表示ロジックが維持されることを CI で担保する。

## テスト対象

- `apps/web/src/components/app/screens/ClothingDetailScreen.tsx`
- `apps/web/scripts/check-clothing-detail-image-layout-spec.mjs`

## テスト観点

1. 服詳細画像が `object-cover` ではなく `object-contain` により全体表示されること。
2. 画像ラッパーが中央寄せ・枠線・背景を持ち、縦長画像でも `max-h-[60dvh]` 以内で収まること。
3. 画像未設定時に既存どおり `no image` プレースホルダを表示すること。
4. 新規テストスクリプトが `package.json` と GitHub Actions CI に登録されること。
5. `resolveImageUrl`、エラー表示、削除済み表示、着た回数、最後に着た日が回帰していないこと。

## テストケース

| ID | 観点 | 確認内容 | 期待結果 |
| --- | --- | --- | --- |
| CDIL-01 | 可変高さ表示 | `ClothingDetailScreen.tsx` の画像 class を確認する | `max-w-full` `h-auto` `max-h-[60dvh]` `object-contain` を使用し、固定高 `object-cover` を使わない |
| CDIL-02 | 画像ラッパー | 画像ラッパーの class を確認する | `flex items-center justify-center rounded-md border border-slate-200 bg-slate-100 p-2` を持つ |
| CDIL-03 | no image フォールバック | 画像未設定時の分岐を確認する | 既存どおり `COMMON_STRINGS.placeholders.noImage` を表示し、プレースホルダの `h-48` を維持する |
| CDIL-04 | CI 配線 | `package.json` と `.github/workflows/ci.yml` を確認する | `test:clothing-detail-image-layout` が scripts に登録され、CI から実行される |
| CDIL-05 | 回帰防止 | 既存の詳細表示ロジックを確認する | `resolveImageUrl`、エラー表示、削除済み表示、着た回数、最後に着た日表示が維持される |

## 実行方法

```bash
pnpm --filter web test:clothing-detail-image-layout
pnpm --filter web test:clothing-detail-screen
pnpm --filter web test:image-url-resolution
```

## CI適用方針

- GitHub Actions の `CI` workflow で `pnpm --filter web test:clothing-detail-image-layout` を実行する。
- 既存の `pnpm --filter web test:clothing-detail-screen` および `pnpm --filter web test:image-url-resolution` とあわせて、服詳細画面の画像表示回帰を検知できる状態を維持する。
