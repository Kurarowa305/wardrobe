# タブバーアイコン表示テスト設計

## 1. 目的

* タブバーの各タブで、文言の上に対応する SVG アイコンが表示されることを静的検証で担保する。
* 現状のアクティブ状態表現を踏襲し、選択中タブのみアクティブ色、それ以外は非アクティブ色の SVG で描画されることを確認する。
* 指定されたSVGコードの path / rect 定義と線スタイルが `TabBarIcon` に正しく反映されることを確認する。
* 再利用可能な SVG コンポーネント化、および旧 PNG アイコン画像の削除を CI 上で継続検証できる状態にする。

## 2. 対象

* `apps/web/src/components/app/navigation/TabBar.tsx`
* `apps/web/src/components/ui/tab-bar-icon.tsx`
* `apps/web/src/app/globals.css`
* `apps/web/scripts/check-tab-bar-icons-spec.mjs`
* `apps/web/scripts/check-tab-bar-icon-svg-spec.mjs`
* `apps/web/package.json`
* `.github/workflows/ci.yml`

## 3. テスト観点

### 3.1 SVG アイコン利用

* `TabBar` が `TabBarIcon` を利用し、4タブのアイコンを SVG で表示していること
* `activeTab` と各タブキーの比較結果を使ってアクティブ状態を `TabBarIcon` に渡していること
* アイコンがタブ文言の上に表示される構造になっていること

### 3.2 再利用可能な SVG 定義

* `components/ui/tab-bar-icon.tsx` にホーム / 履歴 / テンプレート / 服の4種アイコン定義があること
* 4種の SVG が共通のベース実装を通じて再利用可能な形で提供されていること

### 3.3 指定SVGコード反映

* `components/ui/tab-bar-icon.tsx` の各アイコンが指定された `path` / `rect` 定義と一致すること
* SVG ベース実装が `fill="none"`、`strokeWidth="1.5"`、`strokeLinecap="round"`、`strokeLinejoin="round"` を適用すること

### 3.4 旧画像資産の削除

* `public/icons` 配下の旧 PNG アイコン画像が削除されていること
* タブバー実装内に旧 PNG への参照が残っていないこと

### 3.5 レイアウト

* `.tab-item` が縦並び (`flex-direction: column`) になっていること
* アイコンと文言の間隔、およびアイコンサイズが指定されていること

### 3.6 CI組み込み

* `pnpm --filter web test:tab-bar-icons` が `apps/web/package.json` に定義されていること
* GitHub Actions の CI で当該テストが実行されること

## 4. テストケース

| ケースID | 観点 | 条件 | 期待結果 |
| --- | --- | --- | --- |
| TBI-01 | SVG アイコン利用 | `TabBar.tsx` を確認する | `TabBarIcon` が利用され、`activeTab` に応じた状態とともに文言上部へ表示される |
| TBI-02 | 再利用可能な SVG 定義 | `components/ui/tab-bar-icon.tsx` を確認する | 4タブ分の SVG 定義が再利用可能なコンポーネントとして実装されている |
| TBI-03 | 指定SVGコード反映 | `components/ui/tab-bar-icon.tsx` と SVG検証スクリプトを確認する | 4タブの `path` / `rect` と線スタイルが指定SVGコードと一致する |
| TBI-04 | 旧画像資産の削除 | `public/icons` 配下を確認する | 旧 PNG アイコン画像が存在しない |
| TBI-05 | レイアウト | `globals.css` を確認する | タブ項目が縦並びになり、アイコンサイズと間隔が指定されている |
| TBI-06 | CI組み込み | `package.json` と `ci.yml` を確認する | タブバーアイコン検証スクリプトと SVG検証スクリプトが npm script と CI に登録されている |

## 5. 実装方針

* Node.js スクリプト `apps/web/scripts/check-tab-bar-icons-spec.mjs` で利用構造・旧 PNG 資産削除・レイアウトを検証する。
* Node.js スクリプト `apps/web/scripts/check-tab-bar-icon-svg-spec.mjs` で指定SVGコードの `path` / `rect` と線スタイルを検証する。
* PR・push 時の CI で継続的に実行し、タブバーの SVG アイコン表示仕様の退行を防止する。
