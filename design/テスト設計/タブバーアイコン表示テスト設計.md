# タブバーアイコン表示テスト設計

## 1. 目的

* タブバーの4タブに指定されたアイコンが表示され、各文言の上に配置されることを静的検証で担保する。
* アイコンと文言が同一の色ルールでアクティブ状態に連動することを確認し、選択中タブの視認性を維持する。
* 追加した検証スクリプトが CI に組み込まれ、退行を継続的に防止できる状態にする。

## 2. 対象

* `apps/web/src/components/app/navigation/TabBar.tsx`
* `apps/web/src/app/globals.css`
* `apps/web/scripts/check-tab-bar-icon-spec.mjs`
* `apps/web/package.json`
* `.github/workflows/ci.yml`

## 3. テスト観点

### 3.1 アイコン割り当て

* ホームに家アイコンが割り当てられていること
* 履歴に時計アイコンが割り当てられていること
* テンプレートに二重の四角アイコンが割り当てられていること
* 服に半袖Tシャツアイコンが割り当てられていること

### 3.2 レイアウト

* 各タブが縦並びレイアウトであること
* アイコン要素が文言要素より前に描画され、文言の上に表示されること
* 選択中タブに `aria-current="page"` が付与されること

### 3.3 アクティブ状態の色連動

* `tab-item-icon` が `currentColor` を利用し、親要素の文字色を継承すること
* 非アクティブ時は `var(--text-sub)`、アクティブ時は `var(--primary-soft)` に切り替わること
* アクティブ時に背景塗りつぶしへ依存せず、アイコンと文言が同じ色で強調されること

### 3.4 CI組み込み

* `pnpm --filter web test:tab-bar-icon` が `apps/web/package.json` に定義されていること
* GitHub Actions の CI で当該テストが実行されること

## 4. テストケース

| ケースID | 観点 | 条件 | 期待結果 |
| --- | --- | --- | --- |
| TBI-01 | アイコン割り当て | `TabBar.tsx` を確認する | 4タブそれぞれに指定アイコンのコンポーネント定義と割り当てがある |
| TBI-02 | レイアウト | `TabBar.tsx` を確認する | アイコンが文言の上に描画され、選択中タブに `aria-current` が付与される |
| TBI-03 | 色連動 | `globals.css` を確認する | 親要素の `color` にアイコンと文言が連動し、アクティブ時に同色へ切り替わる |
| TBI-04 | CI組み込み | `package.json` と `ci.yml` を確認する | アイコン検証スクリプトが npm script と CI に追加されている |

## 5. 実装方針

* Node.js スクリプト `apps/web/scripts/check-tab-bar-icon-spec.mjs` で、TabBar 実装・スタイル・CI 定義の必要文字列を検証する。
* 既存の固定タブバー検証とは分離し、アイコン仕様の退行のみを独立して検出できるようにする。
