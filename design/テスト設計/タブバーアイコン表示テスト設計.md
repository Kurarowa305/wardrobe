# タブバーアイコン表示テスト設計

## 1. 目的

* タブバーの各タブで、文言の上に対応するアイコン画像が表示されることを静的検証で担保する。
* 現状のアクティブ状態表現を踏襲し、選択中タブのみ active アイコン、それ以外は inactive アイコンへ切り替わることを確認する。
* 追加した検証スクリプトが CI に組み込まれ、退行を防止できる状態にする。

## 2. 対象

* `apps/web/src/components/app/navigation/TabBar.tsx`
* `apps/web/src/app/globals.css`
* `apps/web/scripts/check-tab-bar-icons-spec.mjs`
* `apps/web/package.json`
* `.github/workflows/ci.yml`

## 3. テスト観点

### 3.1 アイコン定義

* ホーム / 履歴 / テンプレート / 服の4タブそれぞれに active / inactive の画像パスが定義されていること
* `public/icons` 配下の画像を利用していること

### 3.2 状態切り替え

* `activeTab` と各タブキーの比較結果を使って active / inactive アイコンを切り替えていること
* アイコン画像がタブ文言の上に表示される構造になっていること

### 3.3 レイアウト

* `.tab-item` が縦並び (`flex-direction: column`) になっていること
* アイコンと文言の間隔、およびアイコンサイズが指定されていること

### 3.4 CI組み込み

* `pnpm --filter web test:tab-bar-icons` が `apps/web/package.json` に定義されていること
* GitHub Actions の CI で当該テストが実行されること

## 4. テストケース

| ケースID | 観点 | 条件 | 期待結果 |
| --- | --- | --- | --- |
| TBI-01 | アイコン定義 | `TabBar.tsx` を確認する | 4タブそれぞれに active / inactive アイコンパスが定義されている |
| TBI-02 | 状態切り替え | `TabBar.tsx` を確認する | `activeTab` に応じて active / inactive アイコンが切り替わり、文言上部に表示される |
| TBI-03 | レイアウト | `globals.css` を確認する | タブ項目が縦並びになり、アイコンサイズと間隔が指定されている |
| TBI-04 | CI組み込み | `package.json` と `ci.yml` を確認する | タブバーアイコン検証スクリプトが npm script と CI に登録されている |

## 5. 実装方針

* Node.js スクリプト `apps/web/scripts/check-tab-bar-icons-spec.mjs` で必要文字列の有無を検証する。
* PR・push 時の CI で継続的に実行し、タブバーのアイコン表示仕様の退行を防止する。
