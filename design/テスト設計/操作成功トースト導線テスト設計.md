# 操作成功トースト導線テスト設計

## 目的

- 追加・編集・削除の各操作後に成功トーストが表示されることを継続的に検証する
- 遷移先画面で query を使ってトーストを一度だけタイトルのみ表示し、表示後に query を消費する実装を CI で担保する
- 服・テンプレート・履歴・ワードローブ作成の各導線で成功通知が統一されていることを確認する

## 対象スクリプト

- `apps/web/scripts/check-operation-toast-spec.mjs`
- 実行コマンド: `pnpm --filter web test:operation-toast`

## テストケース

### OTS-01 操作結果トースト用の query helper が定義されている
- 観点: 成功トースト導線を画面ごとの個別実装ではなく共通 helper で扱えていること
- 期待結果:
  - `src/features/toast/operationToast.ts` に `OPERATION_TOAST_QUERY_KEY` が定義されている
  - `appendOperationToast` と `consumeOperationToast` が定義されている

### OTS-02 服の追加・編集・削除で toast query を付けて遷移する
- 観点: 服操作の成功時に遷移先でトースト表示できること
- 期待結果:
  - 追加時に `OPERATION_TOAST_IDS.clothingCreated` を付与して一覧へ遷移する
  - 編集時に `OPERATION_TOAST_IDS.clothingUpdated` を付与して詳細へ遷移する
  - 削除時に `OPERATION_TOAST_IDS.clothingDeleted` を付与して一覧へ遷移する

### OTS-03 服一覧・詳細画面が追加/編集/削除成功トーストを表示して query を消費する
- 観点: 服画面での成功通知が 1 回だけ表示されること
- 期待結果:
  - 一覧画面で追加成功・削除成功トーストを表示する
  - 詳細画面で編集成功トーストを表示する
  - 表示後に `window.history.replaceState` で query を除去する

### OTS-04 テンプレートの追加・編集・削除で toast query を付けて遷移し、一覧・詳細で成功トーストを表示する
- 観点: テンプレート操作の成功通知導線の一貫性
- 期待結果:
  - 追加時に `templateCreated`、編集時に `templateUpdated`、削除時に `templateDeleted` を利用する
  - 一覧画面で追加成功・削除成功トーストを表示する
  - 詳細画面で編集成功トーストを表示する

### OTS-05 履歴の追加・削除で toast query を付けて遷移し、ホーム/履歴一覧で成功トーストを表示する
- 観点: 履歴は戻り先が複数あるため、遷移先ごとに正しい成功通知を出せること
- 期待結果:
  - テンプレート記録・組み合わせ記録の両画面で `historyCreated` を付与してホームへ遷移する
  - 履歴削除で `historyDeleted` を付与して戻り先へ遷移する
  - ホーム画面で記録追加・履歴削除トーストを表示する
  - 履歴一覧画面でも履歴削除トーストを表示する

### OTS-06 ワードローブ作成も共通 toast query helper を利用する
- 観点: 既存の作成完了通知も同じ仕組みに揃えられていること
- 期待結果:
  - ワードローブ作成画面が `wardrobeCreated` を付与してホームへ遷移する
  - ホーム画面が `wardrobeCreated` を解釈してトースト表示する

### OTS-07 成功トースト文言が各 feature strings に定義されている
- 観点: 文言が画面内にベタ書きされず feature strings に集約されていること
- 期待結果:
  - `clothing/strings.ts` に追加・編集・削除成功文言がある
  - `template/strings.ts` に追加・編集・削除成功文言がある
  - `history/strings.ts` に削除成功文言がある
  - `record/strings.ts` に記録追加成功文言がある

### OTS-08 ホーム画面の成功トーストはタイトルのみを表示する
- 観点: 成功通知の情報量をタイトルに限定し、2行目文言を出さないこと
- 期待結果:
  - `home/strings.ts` の成功トースト定義に `description` が存在しない
  - `HomeTabScreen.tsx` の toast 呼び出しで `description` を渡していない

## CI適用

- `.github/workflows/ci.yml` に `Operation toast spec test` を追加し、PR 時に自動検証する
