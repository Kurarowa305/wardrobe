# カードサムネイルレイアウト統一テスト設計

## 目的

- ホーム、履歴一覧、テンプレート一覧、テンプレートで記録画面のカードサムネイルが共通コンポーネントで統一されることをCIで担保する
- サムネイル上限4枚と、5枠横並びで `+x` まで1列表示できるサイズ設計が維持されることを検証する

## 対象

- `apps/web/src/components/app/shared/ThumbnailRail.tsx`
- `apps/web/src/components/app/history/HistoryCard.tsx`
- `apps/web/src/components/app/screens/TemplatesTabScreen.tsx`
- `apps/web/src/components/app/screens/RecordByTemplateScreen.tsx`
- `apps/web/scripts/check-card-thumbnail-layout-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### CTL-01 共通サムネイルレールコンポーネントが存在する

- 観点: 対象画面のサムネイル実装が共通化され、レイアウト差分が発生しにくいこと
- 確認項目:
  - `ThumbnailRail.tsx` が存在する
  - `items`, `deletedLabel`, `thumbnailAltSuffix`, `limit` を受け取る

### CTL-02 サムネイルは5枠横並びで収まる固定サイズを採用する

- 観点: サムネイル4枚と `+x` 表示を含めて1列表示できるサイズであること
- 確認項目:
  - スロットサイズが `h-10 w-10 sm:h-12 sm:w-12`
  - レールが `flex-nowrap` で折り返さない
  - `overflow-hidden` で横幅超過を抑制する

### CTL-03 サムネイル表示上限は4件で、超過時に `+x` を表示する

- 観点: 仕様どおり4枚までを画像表示し、5枠目に超過件数を表示できること
- 確認項目:
  - `DEFAULT_THUMBNAIL_LIMIT = 4`
  - `items.slice(0, limit)` を使う
  - `+{hiddenCount}` を表示する

### CTL-04 ホーム・履歴一覧の履歴カードが共通サムネイルレールを利用する

- 観点: 履歴カードの見た目がホームと履歴一覧で完全に一致すること
- 確認項目:
  - `HistoryCard.tsx` が `ThumbnailRail` を import する
  - `HISTORY_THUMBNAIL_LIMIT = 4` を維持する

### CTL-05 テンプレート一覧とテンプレートで記録画面が共通サムネイルレールを利用する

- 観点: テンプレート系カードでも同一サイズ・同一 `+x` 表示ルールが適用されること
- 確認項目:
  - `TemplatesTabScreen.tsx` が `ThumbnailRail` を使う
  - `RecordByTemplateScreen.tsx` が `ThumbnailRail` を使う
  - 両画面とも `TEMPLATE_THUMBNAIL_LIMIT = 4` を維持する

## テストスクリプト

- スクリプト: `pnpm --filter web test:card-thumbnail-layout`
- 実体: `apps/web/scripts/check-card-thumbnail-layout-spec.mjs`

## CI適用

- `.github/workflows/ci.yml` に `Card thumbnail layout spec test` を追加し、push時に自動検証する

- CTL-04 履歴カードでの共通利用
- CTL-05 テンプレート一覧 / テンプレート記録画面での共通利用
