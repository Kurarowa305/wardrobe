# カードサムネイルレイアウト統一テスト設計

## 目的

- ホーム、履歴一覧、テンプレート一覧、テンプレートで記録画面のカードサムネイルレイアウトが同一実装で維持されることをCIで担保する
- 5つのサムネイル画像が横並びでちょうど収まるサイズ・列数設定になっていることを継続検証する
- `no image`、`削除済み`、超過件数 `+x` の表示責務が共通コンポーネントに集約されていることを検知する

## 対象

- `apps/web/src/components/app/shared/ThumbnailStrip.tsx`
- `apps/web/src/components/app/history/HistoryCard.tsx`
- `apps/web/src/components/app/screens/TemplatesTabScreen.tsx`
- `apps/web/src/components/app/screens/RecordByTemplateScreen.tsx`
- `apps/web/scripts/check-card-thumbnail-layout-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### CTL-01 共通サムネイルコンポーネントが所定ファイルに存在する
- 観点: カードレイアウト統一の責務が専用コンポーネントに分離されていること

### CTL-02 共通コンポーネントが5件上限・5列グリッドを定義する
- 観点: 5つのサムネイル画像が横並びでちょうど配置しきれるレイアウトになっていること
- 確認項目:
  - `CARD_THUMBNAIL_LIMIT = 5`
  - `grid-cols-5`
  - `slice(0, CARD_THUMBNAIL_LIMIT)`

### CTL-03 共通コンポーネントが no image / 削除済み / +x 表示を担当する
- 観点: 一覧カードごとの差分を減らし、表示仕様を共通化できていること

### CTL-04 ホーム・履歴一覧の履歴カードが共通コンポーネントを利用する
- 観点: 履歴カード表示がホームと履歴一覧で同一レイアウトになること

### CTL-05 テンプレート一覧カードが共通コンポーネントを利用する
- 観点: テンプレート一覧のサムネイル表示が他画面と統一されること

### CTL-06 テンプレート記録画面の選択カードが共通コンポーネントを利用する
- 観点: テンプレート選択カードのサムネイル表示がテンプレート一覧と同一になること

## テスト実装

- スクリプト: `pnpm --filter web test:card-thumbnail-layout`
- 実体: `apps/web/scripts/check-card-thumbnail-layout-spec.mjs`

## CI適用

- `.github/workflows/ci.yml` に `Card thumbnail layout spec test` を追加し、pushごとに自動実行する

## PRサマリーに記載するテストケース

- CTL-01 共通サムネイルコンポーネントの配置
- CTL-02 5件上限・5列グリッド
- CTL-03 `no image` / `削除済み` / `+x` の共通化
- CTL-04 ホーム・履歴一覧の共通履歴カード適用
- CTL-05 テンプレート一覧カードへの適用
- CTL-06 テンプレート記録カードへの適用
