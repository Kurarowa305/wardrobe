# カードサムネイルレイアウト統一テスト設計

## 1. 目的

- ホーム、履歴一覧、テンプレート一覧、テンプレートで記録画面のカード内サムネイルが同一レイアウトで表示されることを確認する。
- サムネイル上限4枚と超過時の `+x` 表示を含め、5枠が1行でちょうど収まるサイズ指定になっていることを確認する。

## 2. 対象

- `apps/web/src/components/app/history/HistoryCard.tsx`
- `apps/web/src/components/app/screens/TemplatesTabScreen.tsx`
- `apps/web/src/components/app/screens/RecordByTemplateScreen.tsx`
- `apps/web/scripts/check-card-thumbnail-layout-spec.mjs`

## 3. テスト観点

1. 履歴カードが5列固定グリッドを使い、サムネイル4枚と `+x` 枠を1行で表示できること。
2. テンプレート一覧カードが履歴カードと同じ5列固定グリッドを採用していること。
3. テンプレートで記録画面の選択カードでも同じ5列固定グリッドを採用し、選択UIと共存していること。
4. 各サムネイルと `+x` 枠が `aspect-square` と `w-full` により正方形で、列幅いっぱいに描画されること。

## 4. テストケース

| ID | 観点 | 期待結果 |
| --- | --- | --- |
| CTL-01 | 履歴カードのサムネイル行 | `grid grid-cols-5 gap-2` を使い、サムネイル・`+x` 枠が1行固定で表示される |
| CTL-02 | テンプレート一覧カードのサムネイル行 | 履歴カードと同じ5列グリッドと正方形サムネイル指定を持つ |
| CTL-03 | テンプレートで記録画面のサムネイル行 | 5列グリッドと正方形サムネイル指定を持ち、選択UIの2カラム構成と両立する |

## 5. 実施方法

- `pnpm --filter web test:card-thumbnail-layout` を実行する。
- スクリプト内で対象ファイルの実装文字列を検査し、5列固定グリッド・正方形サムネイル・`+x` 枠の指定を検証する。
