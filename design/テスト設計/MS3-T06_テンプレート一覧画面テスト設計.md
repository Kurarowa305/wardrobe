# MS3-T06 テンプレート一覧画面テスト設計

## 目的

- MS3-T06（テンプレート一覧画面）の完了条件を継続的に検証する
- テンプレート一覧の主要導線、状態表示、サムネイル表示ルールをCIで担保する

## 対象

- `apps/web/src/components/app/screens/TemplatesTabScreen.tsx`
- `apps/web/src/features/template/strings.ts`
- `apps/web/scripts/check-template-list-screen-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### TLS-01 テンプレート一覧画面が所定ファイルに存在する
- 観点: 一覧画面の実装配置がディレクトリ設計に沿っていること

### TLS-02 useTemplateList と cursor ページングで一覧取得する
- 観点: MS3-T05で作成した Query hook を利用し、一覧の段階的取得に対応していること
- 確認項目:
  - `"use client";` が付与されている
  - `useTemplateList(wardrobeId, { limit, cursor })` を呼ぶ
  - `TEMPLATE_LIST_PAGE_SIZE = 20` を指定する

### TLS-03 「＋ テンプレートを追加」導線が追加画面へ遷移する
- 観点: 一覧画面からテンプレート追加フローに入れること
- 確認項目:
  - `ROUTES.templateNew(wardrobeId)` を利用する
  - `TEMPLATE_STRINGS.list.actions.add` を表示する

### TLS-04 読込中・空状態・読込失敗の表示文言を持つ
- 観点: 一覧画面の基本状態がユーザーに可視化されること
- 確認項目:
  - `loading`
  - `empty`
  - `error`

### TLS-05 テンプレートカードが詳細遷移導線とサムネ表示を持つ
- 観点: カード全体タップで詳細へ遷移し、構成服サムネを表示できること
- 確認項目:
  - `ROUTES.templateDetail(wardrobeId, item.templateId)` を利用する
  - `resolveImageUrl` で画像URLを解決する
  - 画像なし時に `no image` を表示する
  - 削除済み服に `削除済み` オーバーレイを表示する

### TLS-06 サムネイル最大4件と超過分 `+x` を表示する
- 観点: 画面仕様・DB設計のサムネイルルールに一致すること
- 確認項目:
  - `TEMPLATE_THUMBNAIL_LIMIT = 4`
  - `slice(0, TEMPLATE_THUMBNAIL_LIMIT)` を使う
  - 超過件数を `+{hiddenCount}` で表示する

### TLS-07 `nextCursor` がある場合に「さらに読み込む」で追加取得する
- 観点: 一覧APIのページングに応じて継続読み込みできること
- 確認項目:
  - `setNextCursor(data.nextCursor)` を保持する
  - ボタン押下で `setCursor(nextCursor)` を呼ぶ
  - `TEMPLATE_STRINGS.list.actions.loadMore` を表示する

### TLS-08 一覧画面向け文言が template strings に定義される
- 観点: 一覧固有の文言を画面から分離し、他画面と同様に集中管理できること
- 確認項目:
  - `loadMore`
  - `loading`
  - `empty`
  - `error`
  - `deleted`

## テスト実装

- スクリプト: `pnpm --filter web test:template-list-screen`
- 実体: `apps/web/scripts/check-template-list-screen-spec.mjs`
- CI適用:
  - `.github/workflows/ci.yml` に `Template list screen spec test` を追加し、PR時に自動検証する

- TLS-05 詳細遷移 + サムネ/no image/削除済み表示
- TLS-06 サムネ最大4件 + `+x`
- TLS-07 「さらに読み込む」ページング
