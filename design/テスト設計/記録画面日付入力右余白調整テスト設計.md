# 記録画面日付入力右余白調整テスト設計

## 目的

- 記録画面の日付入力欄で、カレンダーアイコン右外側の余白が左側と同等に確保されることを継続的に検証する。
- テンプレート記録画面・服組み合わせ記録画面の両方で同一の余白調整が適用されることを CI で担保する。

## 対象範囲

- `apps/web/src/components/app/screens/RecordByTemplateScreen.tsx`
- `apps/web/src/components/app/screens/RecordByCombinationScreen.tsx`
- `apps/web/scripts/check-record-date-input-spacing-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### RDIS-01 テンプレート記録画面の日付入力欄が右側に左側と同等の余白を持つ
- 観点: カレンダーアイコンが入力欄右端に密着せず、左側の内側余白と視覚的にそろうこと
- 期待値:
  - 日付入力欄に `date-input-balanced` クラスが付与されている
  - `pr-3` で入力欄右側に余白を確保している
  - `::-webkit-calendar-picker-indicator` に `ml-3` を付与してアイコン左側にも間隔を取っている

### RDIS-02 服組み合わせ記録画面の日付入力欄が右側に左側と同等の余白を持つ
- 観点: 服組み合わせ記録画面でも同じ日付入力スタイルが適用され、画面間で見た目がぶれないこと
- 期待値:
  - 日付入力欄に `date-input-balanced` クラスが付与されている
  - `pr-3` で入力欄右側に余白を確保している
  - `::-webkit-calendar-picker-indicator` に `ml-3` を付与してアイコン左側にも間隔を取っている

## テストスクリプト

- スクリプト: `pnpm --filter web test:record-date-input-spacing`
- 実体: `apps/web/scripts/check-record-date-input-spacing-spec.mjs`

## CI適用方針

- GitHub Actions の `CI` workflow に `Record date input spacing spec test` を追加する。
- 既存の記録画面仕様テスト群と同列に配置し、push ごとに継続実行する。

## PRサマリーに記載するテストケース

- RDIS-01 テンプレート記録画面の日付入力欄右余白調整
- RDIS-02 服組み合わせ記録画面の日付入力欄右余白調整
