# 履歴詳細テンプレートカードテスト設計

## 目的
- 履歴詳細画面で、テンプレート入力時にのみ「着用したテンプレート」カードが表示されることをCIで継続検証する。
- 入力方法カードの2行目に旧来の補足文言（テンプレート名/選択した服の組み合わせ）が表示されないことを担保する。

## 対象
- `apps/web/src/components/app/screens/HistoryDetailScreen.tsx`
- `apps/web/src/api/schemas/history.ts`
- `apps/web/src/features/history/types.ts`
- `apps/web/src/features/history/strings.ts`
- `apps/web/src/mocks/fixtures/history.ts`
- `apps/web/src/mocks/handlers/history.ts`
- `apps/web/scripts/check-history-detail-template-card-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点
1. 入力方法カードから補足の2行目文言が削除されている。
2. テンプレート入力時のみ、日付/入力方法カードと着用服一覧の間に「着用したテンプレート」カードが表示される。
3. 着用したテンプレートカードは白背景で、テンプレート名・着用回数・曜日付き最終着用日を着用服カードに近い行構成で表示する。
4. 履歴詳細DTO/VMがテンプレートカード表示に必要な `templateId` / `name` / `wearCount` / `lastWornAt` を保持する。
5. fixture / MSW handler がテンプレート入力履歴にテンプレート詳細情報を同梱する。
6. CI で専用スクリプトが実行される。

## テストケース
- **HDT-01**: 入力方法カードの2行目から旧来の補足文言を削除している。
- **HDT-02**: テンプレート入力時のみ着用テンプレートカードを白背景で表示し、名称・着用回数・曜日付き最終着用日を服カード風の文言で描画する。
- **HDT-03**: 履歴詳細DTO/VMがテンプレートカード表示用の情報を保持する。
- **HDT-04**: fixture/handlerがテンプレート入力履歴にテンプレート詳細情報を含める。
- **HDT-05**: 履歴詳細向け文言に着用テンプレートカード用ラベルが定義される。
- **HDT-06**: CI が `test:history-detail-template-card` を実行する。

## 実行コマンド
- `pnpm --filter web test:history-detail-template-card`
