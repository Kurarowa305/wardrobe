# MS4-T08 履歴削除導線テスト設計

## 目的
- MS4-T08（履歴削除導線）の完了条件を継続的に検証する。
- 履歴詳細からの削除操作が、確認・削除実行・戻り先遷移・失敗通知の仕様どおりに維持されることをCIで担保する。
- TanStack Query設計で定義された履歴削除後のinvalidate範囲（history / clothing / template）を静的検証で固定化する。

## 対象
- `apps/web/src/components/app/screens/HistoryDetailScreen.tsx`
- `apps/web/src/features/history/strings.ts`
- `apps/web/src/api/hooks/history.ts`
- `apps/web/scripts/check-history-delete-flow-spec.mjs`
- `.github/workflows/ci.yml`
- `apps/web/package.json`

## 完了条件との対応
1. 履歴詳細画面のヘッダーメニューから削除操作を実行できる。
2. 削除前に共通削除確認文言を用いた確認ダイアログを表示する。
3. 削除成功後は `from=home|histories` の文脈に応じてホームまたは履歴一覧へ戻る。
4. `from` 未指定または不正値の場合でも履歴一覧へフォールバックできる。
5. 削除失敗時は destructive toast でエラーを通知する。
6. 履歴削除mutation成功後に履歴詳細・履歴一覧・服・テンプレートの関連Queryをinvalidateする。
7. テストスクリプトが `apps/web/package.json` に登録され、CIから実行される。

## テストケース

### HDL-01 履歴詳細画面が履歴削除Mutationを使用する
- 観点: 詳細画面からDELETE APIを実行できること
- 検証方法: `useDeleteHistoryMutation` のimport・hook呼び出し・`mutateAsync` 実行の有無を静的検証する

### HDL-02 削除確認に共通ダイアログ文言を利用する
- 観点: 画面仕様の削除確認文言が共通化され、履歴詳細でも再利用されること
- 検証方法: `COMMON_STRINGS.dialogs.confirmDelete.title/message` と `window.confirm(...)` の利用を確認する

### HDL-03 削除成功時に文脈依存の戻り先へ遷移する
- 観点: ホーム起点・履歴一覧起点の双方で削除後遷移が仕様どおりになること
- 検証方法: `resolveHistoryDetailBackHref(...)` による `backHref` 解決と `router.push(backHref)` の実装を確認する

### HDL-04 削除失敗時にエラートーストを表示する
- 観点: APIエラー時にユーザーへ失敗が通知されること
- 検証方法: `HISTORY_STRINGS.detail.messages.deleteError` の参照、および文言定義の存在を確認する

### HDL-05 ヘッダーメニューの削除項目が削除中無効化に対応する
- 観点: 二重送信防止とメニューアクション連携
- 検証方法: `headerActions` の削除項目に `onSelect: handleDelete` と `deleteMutation.isPending` を用いた `disabled` 制御があることを確認する

### HDL-06 履歴削除後のinvalidate設計が保たれている
- 観点: 履歴削除に伴う履歴・服・テンプレートの表示整合性維持
- 検証方法: `useDeleteHistoryMutation` が履歴詳細invalidateと `invalidateHistoryQueries` を併用していることを確認する

### HDL-07 テストスクリプトが package.json と CI に登録されている
- 観点: 静的検証がローカル・CIの双方で継続実行されること
- 検証方法: `apps/web/package.json` の script 登録と `.github/workflows/ci.yml` の実行ステップ追加をレビューで確認する

## 実行コマンド
- `pnpm --filter web test:history-delete-flow`

- HDL-04 削除失敗時にエラートーストを表示する
- HDL-05 ヘッダーメニューの削除項目が削除中無効化される
- HDL-06 履歴削除後に履歴詳細・履歴一覧・服・テンプレートQueryをinvalidateする
- HDL-07 テストスクリプトがpackage.jsonとCIに登録される
