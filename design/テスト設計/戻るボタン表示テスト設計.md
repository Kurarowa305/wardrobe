# 戻るボタン表示テスト設計

## 目的

- 詳細 / 追加 / 編集 / 記録画面に共通で表示する戻る導線が、最新の文言と軽量なテキストリンク表示へ統一されていることをCIで継続検証する
- 画面仕様にある「左: 戻る」の要件を保ちつつ、外枠を持たない見た目変更が崩れないことを担保する

## 対象スクリプト

- `apps/web/scripts/check-back-button-spec.mjs`
- 実行コマンド: `pnpm --filter web test:back-button`

## テストケース

### BB-01 戻る文言が `&lt; 戻る` に更新されている
- 観点: 共通ナビゲーション文言が指定どおり更新されているか
- 期待結果: `src/constants/navigationStrings.ts` の `NAVIGATION_STRINGS.back` が `< 戻る` を保持する

### BB-02 戻るボタンが共通文言を利用する
- 観点: ヘッダー側で個別文言を直書きせず、共通定義を参照しているか
- 期待結果: `Header.tsx` が `NAVIGATION_STRINGS.back` を `BackButton` に渡している

### BB-03 戻るボタンの外枠が除去されている
- 観点: 外枠付きのピル型ボタンから、テキストリンク風の見た目へ移行できているか
- 期待結果: `globals.css` の `.nav-back-button` に以下が定義されている
  - `justify-content: flex-start;`
  - `padding: 0;`
  - `border: 0;`
  - `background: transparent;`

### BB-04 旧スタイルが残っていない
- 観点: 外枠・角丸・最小幅など旧ボタンスタイルが混在していないか
- 期待結果: `.nav-back-button` に以下が存在しない
  - `min-width: 56px;`
  - `border-radius: 999px;`
  - `border: 1px solid var(--line);`
  - `background: #fff;`

### BB-05 検証スクリプトが package.json と CI に登録されている
- 観点: PR / push 時に自動検証されるか
- 期待結果:
  - `apps/web/package.json` に `test:back-button` が定義されている
  - `.github/workflows/ci.yml` に `pnpm --filter web test:back-button` の実行ステップが存在する
