# MS3-T06 テンプレ一覧画面テスト設計

## 目的

- MS3-T06（テンプレ一覧画面）の完了条件を継続的に検証する
- 一覧表示・空状態・ローディング・追加読み込み・サムネ表示仕様（最大4件 / `+x` / `no image` / 削除済み）をCIで担保する

## 対象スクリプト

- `apps/web/scripts/check-template-list-screen-spec.mjs`
- 実行コマンド: `pnpm --filter web test:template-list-screen`

## テストケース

### TLS-01 テンプレ一覧画面が規定ファイルに存在する
- 観点: 実装配置の統一
- 期待結果: `apps/web/src/components/app/screens/TemplatesTabScreen.tsx` が存在する

### TLS-02 テンプレ一覧画面が useTemplateList を利用し、cursorページングで取得する
- 観点: MS3-T05 hooks再利用とページング要件への準拠
- 期待結果:
  - client component として実装される
  - `useTemplateList(wardrobeId, { limit, cursor })` を利用する
  - `limit` にページサイズ定数を使う

### TLS-03 一覧画面に「＋ テンプレートを追加」導線がある
- 観点: テンプレ追加画面への主要導線
- 期待結果:
  - `TEMPLATE_STRINGS.list.actions.add` を表示する
  - 遷移先が `ROUTES.templateNew(wardrobeId)` である

### TLS-04 読込中・空状態・読込失敗の表示を持つ
- 観点: 状態別表示要件（空/読込）
- 期待結果:
  - `loading` / `empty` / `error` の各文言が画面で利用される

### TLS-05 テンプレカードが詳細遷移し、サムネ仕様（最大4 / +x / no image / 削除済み）を満たす
- 観点: 完了条件「サムネ＋削除済み服の扱い」および一覧カード共通仕様
- 期待結果:
  - 各カードの遷移先が `ROUTES.templateDetail(wardrobeId, item.templateId)` である
  - サムネ表示件数上限が4件である
  - 画像なし時に `COMMON_STRINGS.placeholders.noImage` を表示する
  - 削除済み服に `削除済み` オーバーレイを表示する
  - 5件目以降がある場合に `+x` 表示を出す

### TLS-06 nextCursor がある場合に「さらに読み込む」で追加取得できる
- 観点: 完了条件「nextCursorがあれば追加読み込みできる」
- 期待結果:
  - `nextCursor` の state を保持する
  - APIレスポンスの `data.nextCursor` を state に反映する
  - 「さらに読み込む」押下で `setCursor(nextCursor)` が実行される

### TLS-07 テンプレ一覧画面向け文言が strings に定義される
- 観点: 文言設計ルール（文言の集約）
- 期待結果:
  - `src/features/template/strings.ts` に `loadMore` を定義する
  - `loading` / `empty` / `error` / `deleted` を定義する

## CI適用

- `.github/workflows/ci.yml` に `Template list screen spec test` を追加し、PR時に自動検証する
