# ホームおすすめ服テスト設計

## 目的

- ホーム画面の「＋ 記録する」直下におすすめ服が表示されることをCIで担保する。
- APIクライアント、TanStack Query hook、MSW handler、キャッシュ更新がAPI-18の仕様と整合することを確認する。

## 対象

- `apps/web/src/api/schemas/clothing.ts`
- `apps/web/src/api/endpoints/clothing.ts`
- `apps/web/src/api/hooks/clothing.ts`
- `apps/web/src/api/queryKeys.ts`
- `apps/web/src/components/app/screens/HomeTabScreen.tsx`
- `apps/web/src/components/app/screens/ClothingListCard.tsx`
- `apps/web/src/components/app/screens/ClothingGenreSection.tsx`
- `apps/web/src/mocks/handlers/clothing.ts`
- `apps/web/scripts/check-clothing-recommendation-api-client-spec.mjs`
- `apps/web/scripts/check-home-clothing-recommendations-spec.mjs`
- `apps/web/scripts/check-clothing-recommendation-msw-spec.mjs`

## テストケース

| ID | 観点 | 確認内容 |
| --- | --- | --- |
| HCR-01 | API DTO | おすすめ服レスポンスの `season` / `seasonTagIds` / `items.tops` / `items.bottoms` を扱える |
| HCR-02 | API client | `/wardrobes/{wardrobeId}/recommendations/clothing` をGETする |
| HCR-03 | Query hook | `useClothingRecommendations` が recommendation query key と staleTime を使う |
| HCR-04 | キャッシュ | clothing mutation 後におすすめqueryをinvalidateし、history mutation の clothing scope invalidation でも対象になる |
| HCR-05 | 表示位置 | ホーム画面で「＋ 記録する」直下、直近履歴より前に表示する |
| HCR-06 | 表示状態 | loading / error / empty / list の分岐を持つ |
| HCR-07 | カード表示 | 服一覧と共通のサムネイル + 服名カードを使い、タグ/最終着用日/着用回数を表示しない |
| HCR-08 | 遷移 | おすすめ服カードが服詳細リンクを持つ |
| HCR-09 | MSW | 季節タグ、`season:all`、ACTIVE、最終着用日が古い順、各ジャンル2件のレスポンスを生成する |
| HCR-10 | CI | 3つの専用テストスクリプトがCIで実行される |

## 実行コマンド

```sh
pnpm --filter web test:clothing-recommendation-api-client
pnpm --filter web test:home-clothing-recommendations
pnpm --filter web test:clothing-recommendation-msw
```
