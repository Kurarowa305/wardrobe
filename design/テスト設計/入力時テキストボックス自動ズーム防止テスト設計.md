# 入力時テキストボックス自動ズーム防止テスト設計

## 目的
- スマートフォンブラウザで文字入力を開始した際に、テキストボックスへ自動的にズームインする挙動を防止する。
- 共通 `Input` コンポーネントの文字サイズを 16px 以上に保つことで、入力画面全体の操作性を維持する。

## 対象
- `apps/web/src/components/ui/input.tsx`
- `apps/web/package.json`
- `.github/workflows/ci.yml`

## テスト観点
1. 共通 `Input` コンポーネントが `text-base` を利用していること。
2. `text-sm` のまま残置されておらず、モバイル Safari 等で自動ズームを誘発しないこと。
3. 検証スクリプトが `apps/web/package.json` に登録されていること。
4. 検証スクリプトが GitHub Actions の CI で実行されること。

## テストケース
| ID | 観点 | 確認内容 | 期待結果 |
| --- | --- | --- | --- |
| IFZ-01 | UI実装 | `Input` の className に `text-base` が含まれるか確認する | 16px 相当の文字サイズが指定されている |
| IFZ-02 | UI実装 | `Input` の className に `text-sm text-slate-900` が残っていないか確認する | 自動ズーム要因となる 14px 指定が存在しない |
| IFZ-03 | テスト導線 | `apps/web/package.json` に `test:input-focus-zoom` が定義されているか確認する | ローカル/CI から同一スクリプトを実行できる |
| IFZ-04 | CI適用 | `.github/workflows/ci.yml` に `pnpm --filter web test:input-focus-zoom` があるか確認する | pull request 時に自動で検証される |

## 実行コマンド
- `pnpm --filter web test:input-focus-zoom`
