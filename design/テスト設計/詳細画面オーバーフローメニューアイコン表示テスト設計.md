# 詳細画面オーバーフローメニューアイコン表示テスト設計

## 1. 目的
- 各種詳細画面の右上縦三点リーダーメニューに、操作文言の右側へ指定SVGアイコンが表示されることを担保する。
- 削除アクションの文言とアイコンが赤系カラーで表示されることを担保する。
- 追加した検証スクリプトが package.json と CI に登録され、継続的に検知できることを担保する。

## 2. 対象
- `apps/web/src/components/app/navigation/OverflowMenu.tsx`
- `apps/web/src/components/app/screens/ClothingDetailScreen.tsx`
- `apps/web/src/components/app/screens/TemplateDetailScreen.tsx`
- `apps/web/src/components/app/screens/HistoryDetailScreen.tsx`
- `apps/web/src/app/globals.css`
- `apps/web/scripts/check-overflow-menu-icon-spec.mjs`
- `apps/web/package.json`
- `.github/workflows/ci.yml`

## 3. テスト観点
1. オーバーフローメニューが編集・削除それぞれに対応するSVGアイコンを持つ。
2. メニュー項目が「左: 文言 / 右: アイコン」のレイアウトになっている。
3. 服詳細・テンプレート詳細・履歴詳細の各画面が、各アクションに対して適切なアイコン種別を指定する。
4. 削除アクションに danger tone が指定され、CSSで赤色スタイルが定義されている。
5. 検証スクリプト実行コマンドが `apps/web/package.json` と `.github/workflows/ci.yml` に追加されている。

## 4. テストケース
| ID | 観点 | 確認内容 | 期待結果 |
| --- | --- | --- | --- |
| OMI-01 | SVG表示 | `OverflowMenu.tsx` に編集/削除SVGと右側描画が実装されているか確認する | 指定SVGの path を含み、文言右側にアイコン描画がある |
| OMI-02 | 画面連携 | 各詳細画面の `headerActions` に `icon` 指定があるか確認する | 編集は `edit`、削除は `delete` が設定される |
| OMI-03 | 削除強調 | 各詳細画面の削除 action に `tone: "danger"` があり、CSSに赤色定義があるか確認する | 削除文言とアイコンが赤色化される |
| OMI-04 | 継続検知 | package.json と CI workflow に新テストが登録されているか確認する | ローカル・CIの両方で検証スクリプトを実行できる |

## 5. 実行方法
- `pnpm --filter web test:overflow-menu-icons`
- 必要に応じて関連既存テストとして `pnpm --filter web test:overflow-menu` を併せて実行する。
