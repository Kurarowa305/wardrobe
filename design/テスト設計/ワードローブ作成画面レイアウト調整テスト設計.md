# ワードローブ作成画面レイアウト調整テスト設計

## 1. 目的

- ワードローブ作成画面からヘッダーと `ScreenCard` を除去した新レイアウトを継続的に検証する
- アクセントカラーの大きなタイトルと、その直下の入力フォーム配置をCIで担保する
- 文言定義が画面実装と一致していることを静的テストで保証する

## 2. 対象

- `apps/web/src/components/app/screens/WardrobeCreateScreen.tsx`
- `apps/web/src/components/app/layout/AppLayout.tsx`
- `apps/web/src/features/wardrobe/strings.ts`
- `apps/web/scripts/check-wardrobe-create-screen-spec.mjs`

## 3. テストケース

### WC-01 ヘッダーを非表示にする
- 観点: ワードローブ作成画面で上部ヘッダーが描画されないこと
- 期待結果: `WardrobeCreateScreen.tsx` から `AppLayout` に `showHeader={false}` が渡されている

### WC-02 アクセントカラーの大きなタイトルを表示する
- 観点: 指定文言がアクセントカラー・大きめサイズで表示されること
- 期待結果:
  - `WARDROBE_STRINGS.create.heroTitle` を利用している
  - `text-3xl font-bold leading-tight text-[var(--primary)]` が指定されている

### WC-03 タイトルの直下に入力欄と作成導線を配置する
- 観点: タイトルブロックの直後にフォームが続くレイアウトであること
- 期待結果:
  - タイトルブロックが `grid gap-3`
  - フォームが `grid gap-4`
  - 同一 section 内で順に配置されている

### WC-04 ScreenCard を利用しない
- 観点: 新レイアウトが `ScreenCard` に依存していないこと
- 期待結果: `WardrobeCreateScreen.tsx` に `ScreenCard` の import / JSX 利用が存在しない

### WC-05 文言定義が画面要件と一致する
- 観点: ヒーロータイトルと補助文言が strings に集約されていること
- 期待結果:
  - `heroTitle: "あなたのワードローブを作成しましょう"`
  - `description: "まずはワードローブ名を入力して、管理を始めましょう。"`
  が `src/features/wardrobe/strings.ts` に定義されている

## 4. 実行方法

- 実行コマンド: `pnpm --filter web test:wardrobe-create-screen`
- 実装方式: Node.js による静的検査スクリプト

## 5. CI組み込み

- `.github/workflows/ci.yml` に `Wardrobe create screen spec test` を追加する
- push 時に自動実行し、レイアウト要件の退行を検知する
