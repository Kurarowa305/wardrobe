# MS2-T04 擬似アップロード実装（PUT先に投げる形だけ再現）テスト設計

## 目的

presign取得後の擬似アップロード（`PUT uploadUrl`）実装、およびアップロード中状態・失敗時リトライ導線を静的検査で担保する。

## 対象

- `apps/web/src/api/endpoints/image.ts`
- `apps/web/src/components/app/screens/ClothingCreateScreen.tsx`
- `apps/web/src/components/app/screens/ClothingEditScreen.tsx`
- `apps/web/src/mocks/handlers/image.ts`
- `apps/web/src/features/clothing/strings.ts`
- `apps/web/scripts/check-image-pseudo-upload-spec.mjs`

## テスト観点

1. presign後に `fetch(uploadUrl, { method: "PUT", body: file })` を実行すること
2. 服追加画面でアップロード中状態を管理し、送信前に擬似アップロードを行うこと
3. 服編集画面でアップロード中状態を管理し、送信前に擬似アップロードを行うこと
4. 追加/編集画面でアップロード失敗時に再試行導線があること
5. MSWが `uploadUrl` へのPUTを受け取り、共通シナリオ（`delay`/`forceError`）を適用できること
6. アップロード中/失敗/再試行の文言が定義されていること

## テストケース

### IPU-01: 擬似アップロード用APIクライアントが image endpoint に存在する

- チェック内容
  - `uploadImageWithPresign` の公開
  - `getPresignedUrl` 実行
  - `fetch(presigned.uploadUrl, { method: "PUT", body: file })` 実行
- 期待結果
  - presign -> PUT のフローが実装されている

### IPU-02: 服追加画面がアップロード中stateを管理し、送信前に擬似アップロードを実行する

- チェック内容
  - `isUploadingImage` / `uploadError` state の存在
  - 送信前に `uploadImageWithPresign(..., "clothing", file)` を実行
  - アップロード中文言を表示できる
- 期待結果
  - 服追加時に擬似アップロード処理を挟める

### IPU-03: 服追加画面がアップロード失敗時に再試行導線を表示する

- チェック内容
  - `handleRetryUpload` の存在
  - アップロード失敗文言の表示
  - 再試行ボタンの表示
- 期待結果
  - 失敗後に再試行できる

### IPU-04: 服編集画面がアップロード中stateを管理し、送信前に擬似アップロードを実行する

- チェック内容
  - `isUploadingImage` / `uploadError` state の存在
  - 送信前に `uploadImageWithPresign(..., "clothing", file)` を実行
  - アップロード中文言を表示できる
- 期待結果
  - 服編集時に擬似アップロード処理を挟める

### IPU-05: 服編集画面がアップロード失敗時に再試行導線を表示する

- チェック内容
  - `handleRetryUpload` の存在
  - アップロード失敗文言の表示
  - 再試行ボタンの表示
  - 送信ボタンが `isNameEmpty || isGenreEmpty || isPending` を条件に無効化され、編集画面の必須入力条件と競合しないこと
- 期待結果
  - 失敗後に再試行でき、ジャンル未選択を含む現行バリデーションと整合する

### IPU-06: MSW image handler が presign uploadUrl への PUT を受け、共通シナリオを適用できる

- チェック内容
  - `http.put(${MOCK_UPLOAD_ORIGIN}/upload/:category/:wardrobeId/:fileName)` の存在
  - `applyMockScenario(request)` の適用
  - empty body の 400 応答
- 期待結果
  - 擬似アップロード先の成功/失敗/遅延を再現できる

### IPU-07: アップロード中/失敗/再試行の文言が clothing strings に定義される

- チェック内容
  - `retryUpload` / `uploadingImage` / `uploadError` の定義
- 期待結果
  - UI状態に応じた文言を表示できる
