# 戻るボタン文言スタイル調整テスト設計

## 目的
- ヘッダー左上の戻るボタンが最新UI仕様どおりに「< 戻る」表記で表示されることを継続的に検証する。
- 戻るボタンの外枠が削除され、フォントサイズが 15px に調整されていることを静的検査で担保する。
- 検証スクリプトが package.json と CI に登録され、ローカル・CI の双方で自動実行されることを確認する。

## 対象
- `apps/web/src/constants/navigationStrings.ts`
- `apps/web/src/components/app/navigation/BackButton.tsx`
- `apps/web/src/app/globals.css`
- `apps/web/package.json`
- `.github/workflows/ci.yml`
- `apps/web/scripts/check-back-button-spec.mjs`

## テスト観点
1. 戻るボタンの文言が「< 戻る」に統一されている。
2. 戻るボタンが `nav-back-button` クラスで描画され、ヘッダー共通導線として利用される。
3. `.nav-back-button` の外枠と背景が削除され、`font-size: 15px;` が指定されている。
4. 検証スクリプトが npm script と CI workflow の双方に登録されている。

## テストケース
| ID | テストケース | 確認内容 | 期待結果 |
| --- | --- | --- | --- |
| BB-01 | 文言更新 | `navigationStrings.ts` の `back` 文言を確認する | `back: "< 戻る"` が定義されている |
| BB-02 | コンポーネント適用 | `BackButton.tsx` の描画内容を確認する | `nav-back-button` クラスの `Link` に `label` が表示される |
| BB-03 | スタイル更新 | `globals.css` の `.nav-back-button` 定義を確認する | `border: 0;` `background: transparent;` `padding: 0;` `font-size: 15px;` が設定されている |
| BB-04 | 自動実行導線 | `package.json` と `ci.yml` を確認する | `test:back-button` が scripts と CI step の両方に登録されている |

## 実行方法
- `pnpm --filter web test:back-button`

## 備考
- 視覚仕様は静的検査で担保し、必要に応じて手動の画面確認を補助的に行う。
