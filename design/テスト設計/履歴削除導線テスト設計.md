# 履歴削除導線テスト設計

## 1. 目的

* MS4-T08 の履歴削除導線（詳細→削除→戻り/更新）を継続的に検証する
* 履歴詳細画面での削除確認、削除成功後の戻り先、削除失敗時の通知を CI で早期検知する

---

## 2. 対象

* `apps/web/src/components/app/screens/HistoryDetailScreen.tsx`
* `apps/web/src/features/history/strings.ts`
* `apps/web/src/features/history/routing.ts`

---

## 3. 実行方法

* ローカル: `pnpm --filter web test:history-delete-flow`
* CI: `.github/workflows/ci.yml` の `History delete flow spec test` ステップで実行
* 実装スクリプト: `apps/web/scripts/check-history-delete-flow-spec.mjs`

---

## 4. テスト項目一覧

| ID | 観点 | 検証内容 |
| --- | --- | --- |
| HDL-01 | Mutation連携 | `useDeleteHistoryMutation(wardrobeId, historyId)` を利用し、削除実行で `mutateAsync()` を呼ぶ |
| HDL-02 | 削除確認 | 共通文言 `COMMON_STRINGS.dialogs.confirmDelete` を使って確認ダイアログを表示する |
| HDL-03 | 成功時遷移 | `from` クエリで解決した `backHref` へ戻る |
| HDL-04 | 失敗時通知 | 削除失敗時に `HISTORY_STRINGS.detail.messages.deleteError` を destructive toast で通知する |
| HDL-05 | メニュー状態 | ヘッダーメニューの削除項目が `onSelect` を持ち、削除中は無効化される |

---

## 5. 運用ルール

* 履歴削除導線を変更した場合は、本資料と `check-history-delete-flow-spec.mjs` を同時更新する
* 履歴詳細の戻り先仕様（`from=home` / `from=histories`）を変更した場合は、`design/テスト設計/ルーティングテスト設計.md` もあわせて更新する
* 削除失敗時のユーザー通知文言を変更した場合は、`apps/web/src/features/history/strings.ts` とテストスクリプトの期待値を一致させる
