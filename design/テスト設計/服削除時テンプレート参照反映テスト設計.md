# 服削除時テンプレート参照反映テスト設計

## 目的

- 服削除後、テンプレート一覧サムネイルとテンプレート詳細の参照文脈で削除済み表記が反映されることを継続的に検証する
- Clothing削除とTemplate参照表示のMSW連携退行をCIで検知できるようにする

## 対象スクリプト

- `apps/web/scripts/check-template-clothing-reference-spec.mjs`
- 実行コマンド: `pnpm --filter web test:template-clothing-reference`

## テストケース

### TCR-01 Clothing handler が現在の服状態を参照用スナップショットとして公開する
- 観点: 服削除後の状態を他ドメイン（Template）が再利用できること
- 期待結果:
  - `getClothingSnapshotById(clothingId)` を export する
  - `clothingStore` から現在値を取得する
  - 呼び出し側へコピーを返し、副作用を防ぐ

### TCR-02 Template handler が fixture 固定ではなく clothing handler の現在状態から構成服を解決する
- 観点: テンプレート参照が初期fixtureに固定されず、削除操作後の服状態を反映できること
- 期待結果:
  - `src/mocks/handlers/template.ts` が `getClothingSnapshotById` を利用する
  - `resolveClothingItems` が現在の服状態から構成服を再解決する
  - fixture辞書への直接依存を除去する

### TCR-03 Template handler が一覧・詳細返却前に構成服状態を同期し、削除済み表記へ反映できる
- 観点: テンプレート一覧サムネイルとテンプレート詳細の両方で削除済み表示に必要な `status` が最新化されること
- 期待結果:
  - `syncTemplateClothingItems` を定義する
  - 一覧返却前に `map(syncTemplateClothingItems)` を行う
  - 詳細返却前に `syncTemplateClothingItems(template)` を適用する

## CI適用

- `.github/workflows/ci.yml` に `Template clothing reference spec test` を追加し、PR時に自動検証する

## PRサマリーに記載するテストケース

- TCR-01 服状態スナップショット公開
- TCR-02 Template参照の解決元を現在の服状態へ切り替え
- TCR-03 一覧/詳細返却前の構成服同期
