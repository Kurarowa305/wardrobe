# MS3-T07 テンプレート追加/編集画面テスト設計

## 目的

- MS3-T07（テンプレート追加/編集画面の服複数選択）の完了条件をCIで継続検証する
- `TemplateForm` を共通利用しつつ、追加/編集の両画面でテンプレート名入力・服複数選択・保存導線が実装されていることを担保する
- 追加/編集画面ではヘッダーの戻る導線に統一し、フォーム内にキャンセルボタンを表示しないことを担保する
- 服一覧取得時に API の `limit` 上限へ配慮した実装であることを担保する

## 対象スクリプト

- `apps/web/scripts/check-template-form-screen-spec.mjs`
- 実行コマンド: `pnpm --filter web test:template-form-screen`

## テストケース

### TF-01 TemplateCreateScreen と TemplateEditScreen が TemplateForm を利用している
- 観点: 追加/編集画面のUI構成をフォーム基盤へ統一できているか
- 期待結果:
  - `TemplateCreateScreen.tsx` が `TemplateForm` を import し、`createElement(TemplateForm, ...)` で描画している
  - `TemplateEditScreen.tsx` が `TemplateForm` を import し、`createElement(TemplateForm, ...)` で描画している

### TF-02 TemplateForm がテンプレート名入力と服複数選択UIを提供している
- 観点: MS3-T07 の主要入力要件を満たしているか
- 期待結果:
  - `TemplateForm.tsx` で `Input` を利用したテンプレート名入力がある
  - `type="checkbox"` による服複数選択UIがある
  - 選択状態を `selectedClothingIds` で保持している
  - 未選択時のエラー文言として `TEMPLATE_STRINGS.messages.clothingRequired` を利用している

### TF-03 TemplateForm は create/update mutation を使い分け、送信成功後に遷移する
- 観点: 追加/編集の送信処理が正しい API と遷移先に接続されているか
- 期待結果:
  - `useCreateTemplateMutation` と `useUpdateTemplateMutation` を利用している
  - 追加時は `createMutation.mutateAsync` 成功後にテンプレート一覧へ遷移する
  - 編集時は `updateMutation.mutateAsync` 成功後にテンプレート詳細へ遷移する

### TF-04 服一覧取得は API の limit 上限を意識した定数指定と追加読み込みに対応している
- 観点: 服一覧取得時の API 制約と将来的な件数増加に対応できているか
- 期待結果:
  - `TemplateForm.tsx` に `TEMPLATE_FORM_CLOTHING_LIMIT = 50` が定義されている
  - `useClothingList` 呼び出しで `limit: TEMPLATE_FORM_CLOTHING_LIMIT` を指定している
  - `nextCursor` を用いた追加読み込みで次ページ取得ができる
  - 追加読み込みボタン文言として `TEMPLATE_STRINGS.actions.loadMoreClothings` を利用している

### TF-05 Template strings にフォーム用の入力・エラー・服一覧文言が定義されている
- 観点: 文言の集約ルールに沿っているか
- 期待結果:
  - `features/template/strings.ts` に `nameRequired` / `clothingRequired` / `loadMoreClothings` / `submitError` が定義されている

### TF-06 TemplateForm はキャンセルボタンを表示しない
- 観点: 追加/編集画面の操作導線をヘッダーの戻る操作に統一できているか
- 期待結果:
  - `TemplateForm.tsx` に `TEMPLATE_STRINGS.actions.cancel` の参照が存在しない
  - `variant="outline"` のキャンセル用ボタンが存在しない
  - `router.push(backHref)` によるフォーム内キャンセル遷移が存在しない

## CI適用

- `.github/workflows/ci.yml` に `Template form screen spec test` を追加し、PR時に自動検証する
