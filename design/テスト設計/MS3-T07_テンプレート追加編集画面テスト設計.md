# MS3-T07 テンプレート追加/編集画面テスト設計

## 目的

- MS3-T07（テンプレート追加/編集画面）の完了条件を継続的に検証する
- テンプレート名入力、服複数選択、作成/更新導線の主要仕様をCIで担保する

## 対象

- `apps/web/src/components/app/screens/TemplateCreateScreen.tsx`
- `apps/web/src/components/app/screens/TemplateEditScreen.tsx`
- `apps/web/src/components/app/screens/TemplateForm.tsx`

- `apps/web/scripts/check-template-form-screens-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### TFS-01 追加/編集画面と共通フォームが所定ファイルに存在する
- 観点: 画面実装と共通フォーム化がディレクトリ設計に沿っていること

### TFS-02 追加画面が服一覧取得と create mutation を利用し保存成功後に一覧へ戻る
- 観点: MS1-T05 の服一覧データを活用し、テンプレート作成フローが成立すること
- 確認項目:
  - `useClothingList(wardrobeId, { limit: TEMPLATE_FORM_CLOTHING_LIMIT })`
  - `useCreateTemplateMutation(wardrobeId)`
  - `router.push(ROUTES.templates(wardrobeId))`

### TFS-03 追加画面がテンプレ名必須・服1件以上必須で `clothingIds` を送信する
- 観点: API設計どおり `name` と `clothingIds[]` を入力し、空送信を防げること
- 確認項目:
  - `selectedClothingIds` state
  - `trimmedName.length === 0 || selectedClothingIds.length === 0`
  - `clothingIds: selectedClothingIds`
  - checkbox の複数選択UI

### TFS-04 編集画面が既存値を反映し update mutation 後に詳細へ戻る
- 観点: 追加画面流用方針に沿って、初期値と実行APIのみ差分化されていること
- 確認項目:
  - `useTemplate(wardrobeId, templateId)`
  - `useUpdateTemplateMutation(wardrobeId, templateId)`
  - `setName(templateQuery.data.name)`
  - `setSelectedClothingIds(...)`
  - `router.push(ROUTES.templateDetail(wardrobeId, templateId))`

### TFS-05 編集画面が404・一般エラー・服一覧取得失敗を表示できる
- 観点: 初期値取得前提の編集画面で必要なエラー導線があること
- 確認項目:
  - 404時に `TEMPLATE_STRINGS.detail.messages.notFound`
  - 一般エラー時に `TEMPLATE_STRINGS.edit.messages.loadError`
  - 服一覧エラー時に `TEMPLATE_STRINGS.edit.messages.clothingLoadError`

### TFS-06 共通フォームがテンプレ名入力、服複数選択、空状態、送信エラー表示を持つ
- 観点: 追加/編集で共通利用するフォームとして基本UIと状態表示を持つこと
- 確認項目:
  - テンプレート名入力
  - 服複数選択 checkbox
  - 服空状態表示
  - 構成服必須エラー
  - submit error 表示

### TFS-07 追加/編集/詳細向け文言が `template strings` に定義される
- 観点: 画面文言が実装から分離され、将来の差し替えが容易であること
- 確認項目:
  - `nameRequired`
  - `clothingRequired`
  - `submitError`
  - `emptyClothing`
  - `notFound`

## テスト実装

- スクリプト: `pnpm --filter web test:template-form-screens`
