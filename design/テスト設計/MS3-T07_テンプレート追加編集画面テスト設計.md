# MS3-T07 テンプレート追加編集画面テスト設計

## 目的
- テンプレート追加/編集画面が共通フォームを使い、服一覧をジャンル別に安定表示できることをCIで担保する
- 服一覧取得が cursor なし・上限100件の仕様へ揃っていることを固定化する

## 対象スクリプト
- `apps/web/scripts/check-template-form-screen-spec.mjs`
- 実行コマンド: `pnpm --filter web test:template-form-screen`

## テストケース
### TF-01 TemplateCreateScreen と TemplateEditScreen が TemplateForm を利用する
- 期待結果: 両画面が `TemplateForm` を利用する

### TF-02 TemplateForm がテンプレート名入力とジャンル別服複数選択UIを提供する
- 期待結果: 入力欄・複数選択状態・`ClothingGenreSection` が揃う

### TF-03 TemplateForm は create/update mutation を使い分け、送信成功後に遷移する
- 期待結果: create/edit それぞれの mutation と遷移先を使い分ける

### TF-04 服一覧取得はジャンル別 limit=100 指定で cursor や追加読み込みを持たない
- 期待結果:
  - `TEMPLATE_FORM_CLOTHING_LIMIT = 100` を使う
  - ジャンル別に `useClothingList` を呼ぶ
  - `handleLoadMore` や `loadMoreClothings` を使わない

### TF-05 Template strings に clothingSectionEmpty を定義する
- 期待結果: 空状態文言が集約される

### TF-06 TemplateForm はキャンセルボタンを描画せず固定保存ボタンを表示する
- 期待結果: 下部固定の保存ボタンを持つ

### TF-07 各セクションが見出し・折りたたみ・同ジャンルアイコンを持つ
- 期待結果: セクション見出しとアイコンが表示される

## CI適用
- `.github/workflows/ci.yml` の `Template form screen design sync spec test` で自動検証する
