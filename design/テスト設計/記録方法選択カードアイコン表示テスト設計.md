# 記録方法選択カードアイコン表示テスト設計

## 1. 目的

* 記録方法選択画面の各カード左上に、タブバーと同一のアイコンが表示されることを静的検証で担保する。
* テンプレートで記録カードはテンプレートタブのアイコン、服の組み合わせで記録カードは服タブのアイコンを利用することを継続検証できる状態にする。
* カード内アイコンの線色が黒で描画されること、および本検証が CI に組み込まれていることを保証する。

## 2. 対象

* `apps/web/src/components/app/screens/RecordMethodScreen.tsx`
* `apps/web/src/components/ui/tab-bar-icon.tsx`
* `apps/web/src/app/globals.css`
* `apps/web/scripts/check-record-method-card-icons-spec.mjs`
* `apps/web/package.json`
* `.github/workflows/ci.yml`

## 3. 検証観点

### 3.1 アイコン再利用

* 記録方法選択画面が `TabBarIcon` を import していること
* テンプレートで記録カードに `templates` アイコンを利用していること
* 服の組み合わせで記録カードに `clothings` アイコンを利用していること

### 3.2 レイアウト

* カード内部が上からアイコン、見出し、説明文の順で並ぶ構造になっていること
* アイコン用クラスに固定サイズが定義されていること

### 3.3 線色制御

* `TabBarIcon` が任意の線色上書きを受け付けること
* 記録方法選択画面から黒（`#000000`）を指定していること

### 3.4 CI 組み込み

* `pnpm --filter web test:record-method-card-icons` が `apps/web/package.json` に定義されていること
* GitHub Actions の CI で同スクリプトが実行されること

## 4. テストケース

| ID | 観点 | 手順 | 期待結果 |
| --- | --- | --- | --- |
| RMI-01 | アイコン再利用 | `RecordMethodScreen.tsx` を確認する | `TabBarIcon` が import され、`templates` と `clothings` が各カードに使われている |
| RMI-02 | 左上表示 | `RecordMethodScreen.tsx` と `globals.css` を確認する | カードが縦配置で、アイコンが各カードの左上に表示される構造とスタイルがある |
| RMI-03 | 黒線表示 | `RecordMethodScreen.tsx` と `tab-bar-icon.tsx` を確認する | `strokeColor` を受け取り、カード側から `#000000` が指定されている |
| RMI-04 | CI 組み込み | `package.json` と `ci.yml` を確認する | 検証スクリプトが npm script と CI の両方に登録されている |

## 5. 実施方法

* Node.js スクリプト `apps/web/scripts/check-record-method-card-icons-spec.mjs` で構造・色指定・スタイル・CI 組み込みを検証する。
* 既存の `check-record-method-screen-spec.mjs` とは責務を分離し、画面存在や導線確認とは独立してアイコン表示仕様の退行を検知する。
* PR・push 時の CI で継続実行し、記録方法選択カードのアイコン表示仕様を保護する。
