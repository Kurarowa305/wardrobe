# MS5-T03 服組合せ記録画面テスト設計

## 目的

- MS5-T03（記録：服組合せで記録）の完了条件を継続的に検証する
- 複数の服を選択して履歴作成できる導線・入力・遷移をCIで担保する
- 服一覧のサムネイル付き選択UIと固定アクションの見た目崩れを検知する

## 対象範囲

- `apps/web/src/components/app/screens/RecordByCombinationScreen.tsx`
- `apps/web/src/features/record/strings.ts`
- `apps/web/scripts/check-record-combination-screen-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### RCS-01 服組合せ記録画面が所定ファイルに存在する
- 観点: 記録フローの組合せ記録画面が設計どおり専用 Screen として存在すること

### RCS-02 AppLayout でタイトルと戻り先が記録方法選択に設定される
- 観点: スタック画面としてタイトルと戻る導線が正しく構成されていること
- 期待値:
  - `RECORD_STRINGS.byCombination.title` を画面タイトルに利用する
  - `ROUTES.recordMethod(wardrobeId)` を戻り先に利用する

### RCS-03 日付入力欄が date input と必須エラーメッセージ付きで実装される
- 観点: 履歴作成時に日付が必須入力として扱われること
- 期待値:
  - `type="date"` の入力欄を持つ
  - 空入力時に `dateRequired` 文言を表示できる

### RCS-04 服一覧取得に clothing query hook を用い、複数選択 checkbox を描画する
- 観点: 服一覧を取得して複数選択 UI を提供できること
- 期待値:
  - `useClothingList(wardrobeId, ...)` を利用する
  - 各服を checkbox で選択できる

### RCS-05 服一覧がサムネイル付き選択リストとして描画される
- 観点: 選択中セクションを設けず、一覧自体にサムネイルと選択状態を集約していること
- 期待値:
  - 各服に左寄せのサムネイルを表示する
  - 各服に右寄せのチェック表示を持つ
  - 選択状態に応じてアクセントカラー系の背景色へ変化する

### RCS-06 記録ボタンが create history mutation を呼び出し、API仕様の yyyymmdd に変換した服ID配列で履歴を作成する
- 観点: 服組合せ記録として履歴作成 API に正しい payload を渡せること
- 期待値:
  - `useCreateHistoryMutation(wardrobeId)` を利用する
  - 日付入力値を `yyyymmdd` に正規化してから `mutateAsync` を呼ぶ
  - `clothingIds: selectedClothingIds` を渡す

### RCS-07 記録ボタンが画面下部固定で表示される
- 観点: スクロール位置に関わらず主要アクションへ到達できること
- 期待値:
  - 送信ボタンを `fixed` で下部表示する
  - リスト末尾がボタンに隠れないよう余白を持つ

### RCS-08 服組合せ記録向け文言が record strings に定義される
- 観点: 画面固有文言が `record/strings.ts` に集約され、不要になった選択中セクション文言が削除されていること
- 期待値:
  - タイトル、ラベル、必須/失敗文言が定義されている
  - `選択中の服` 文言が残っていない

## テストスクリプト

- スクリプト: `pnpm --filter web test:record-combination-screen`
- 実体: `apps/web/scripts/check-record-combination-screen-spec.mjs`

## CI適用方針

- GitHub Actions の `CI` workflow に `Record combination screen spec test` を追加する
- 既存の画面仕様テスト群と同列に配置し、push ごとに継続実行する

## PRサマリーに記載するテストケース

- RCS-01 服組合せ記録画面のファイル配置
- RCS-02 AppLayout のタイトル / 戻り導線
- RCS-03 日付入力と必須バリデーション
- RCS-04 服一覧取得と複数選択 UI
- RCS-05 服一覧のサムネイル付き選択リスト
- RCS-06 yyyymmdd 変換を含む履歴作成 mutation の payload
- RCS-07 下部固定の記録ボタン
- RCS-08 文言定義の配置と不要文言削除
