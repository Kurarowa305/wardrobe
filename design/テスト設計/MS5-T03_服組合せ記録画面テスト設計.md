# MS5-T03 服組合せ記録画面テスト設計

## 目的
- 服組合せ記録画面の入力・選択・送信導線をCIで担保する
- 服一覧取得が cursor なし・上限100件仕様へ揃っていることを固定化する

## 対象スクリプト
- `apps/web/scripts/check-record-combination-screen-spec.mjs`
- 実行コマンド: `pnpm --filter web test:record-combination-screen`

## テストケース
### RCS-01 服組み合わせ記録画面が存在する
- 期待結果: `RecordByCombinationScreen.tsx` が存在する

### RCS-02 AppLayout でタイトルと戻り先が設定される
- 期待結果: タイトルと戻り先が定義される

### RCS-03 日付入力欄が date input と必須エラー付きで実装される
- 期待結果: date input と必須エラー文言を持つ

### RCS-04 服一覧取得にジャンル別 clothing query hook を用い、複数選択 checkbox を描画する
- 期待結果: ジャンル別 `useClothingList` と checkbox UI を持つ

### RCS-05 服一覧がジャンル別サムネイル付き選択セクションとして描画される
- 期待結果: `ClothingGenreSection` によるセクション描画を行う

### RCS-06 記録ボタンが create history mutation を呼び、yyyymmdd に変換した服ID配列で履歴を作成する
- 期待結果: 送信payloadが `date` と `clothingIds` を含む

### RCS-07 ジャンルごとに limit=100 で取得し「さらに読み込む」を持たない
- 期待結果:
  - `RECORD_COMBINATION_CLOTHING_LIMIT = 100` を使う
  - `handleLoadMore` や `loadMoreLabel` を持たない

### RCS-08 record strings に sectionEmpty を定義する
- 期待結果: `sectionEmpty` と `clothingRequired` を持つ

## CI適用
- `.github/workflows/ci.yml` の `Record combination screen spec test` で自動検証する
