# MS4-T09 最終着用日曜日表示テスト設計

## 目的
- 最終着用日を表示する全画面で、日付が曜日付き `YYYY/MM/DD (曜)` 形式で統一されることをCIで担保する。

## 対象
- `apps/web/src/lib/date.ts`
- `apps/web/src/components/app/screens/ClothingDetailScreen.tsx`
- `apps/web/src/components/app/screens/TemplateDetailScreen.tsx`
- `apps/web/src/components/app/screens/HistoryDetailScreen.tsx`
- `apps/web/scripts/check-last-worn-at-weekday-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点
1. 共通日付ユーティリティがタイムスタンプを曜日付き `YYYY/MM/DD (曜)` に整形する。
2. 服詳細画面が共通ユーティリティを使い、最終着用日を曜日付き表示する。
3. テンプレート詳細画面がテンプレ本体・構成服の最終着用日を曜日付き表示する。
4. 履歴詳細画面が着用テンプレート・着用服の最終着用日を曜日付き表示する。
5. 専用テストスクリプトが package script と CI に登録され、PR時に自動実行される。

## テストケース
- **LWW-01**: `formatTimestampDateWithWeekday` が曜日付き `YYYY/MM/DD (曜)` を返す。
- **LWW-02**: `ClothingDetailScreen.tsx` が `formatTimestampDateWithWeekday` を利用する。
- **LWW-03**: `TemplateDetailScreen.tsx` が `formatTimestampDateWithWeekday` を利用する。
- **LWW-04**: `HistoryDetailScreen.tsx` が `formatTimestampDateWithWeekday` を利用する。
- **LWW-05**: `apps/web/package.json` と `.github/workflows/ci.yml` に `test:last-worn-at-weekday` が登録される。

## 実行コマンド
- `pnpm --filter web test:last-worn-at-weekday`
