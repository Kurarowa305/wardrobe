# 履歴記録テンプレート解決ガードテスト設計

## 目的
- 履歴記録 API (`POST /wardrobes/{wardrobeId}/histories`) のテンプレート参照時に、`GetItem` 返却形式（`Item` / `item`）の揺れを吸収して正しく `clothingIds` を解決できることを継続検証する。
- テンプレート未存在・非ACTIVE・`clothingIds` 不正を 500 ではなく `NOT_FOUND` / `VALIDATION_ERROR` で返し、既存の handler 例外正規化と共存できることを CI で再発防止する。

## 対象
- `apps/api/src/domains/history/usecases/createHistoryWithStatsWrite.ts`
- `apps/api/src/domains/history/handlers/createHistoryHandler.ts`
- `apps/api/scripts/check-history-template-source-guard-spec.mjs`
- `apps/api/package.json`
- `.github/workflows/ci.yml`

## テスト観点
1. テンプレート取得結果の `Item` / `item` 揺れを吸収して `clothingIds` を取得できること
2. テンプレート未存在、または `status !== ACTIVE` の場合に `NOT_FOUND` を返すこと
3. テンプレートに `clothingIds` が欠落・空配列・配列要素型不整合の場合に `VALIDATION_ERROR` を返すこと
4. `createHistoryHandler` が既存の `TransactionCanceledException -> NOT_FOUND` 正規化を保持しつつ、usecase 由来 `NOT_FOUND` / `VALIDATION_ERROR` を透過すること
5. 追加テストが package script と CI の双方へ接続されること

## テストケース
| ID | 観点 | 条件 | 期待結果 |
| --- | --- | --- | --- |
| HTSG-01 | `Item` / `item` 揺れ吸収 | `getTemplate` が `item.status=ACTIVE` と `item.clothingIds` を返す | 履歴作成が成功し `historyId` を返す |
| HTSG-02 | テンプレ未存在/非ACTIVE | `getTemplate` が `{ item: undefined }` または `status=DELETED` を返す | usecase が `NOT_FOUND` を送出する |
| HTSG-03 | `clothingIds` 欠落・空配列・型不整合 | `getTemplate` が `clothingIds` 不正な `Item` を返す | usecase が `VALIDATION_ERROR` を送出する |
| HTSG-04 | handler 例外透過 | `createHistoryHandler` 経由で usecase が `NOT_FOUND` / `VALIDATION_ERROR` を送出する | 同じ `code` がそのまま返る |
| HTSG-05 | 既存正規化との共存 | `createHistoryHandler` ソース上で `TransactionCanceledException` 判定が維持される | 既存正規化ロジックが残っている |
| HTSG-06 | CI導線 | `package.json` / `ci.yml` を検査する | `test:history-template-source-guard` が scripts と CI に登録される |

## 実行コマンド
- `pnpm --filter api test:history-template-source-guard`
