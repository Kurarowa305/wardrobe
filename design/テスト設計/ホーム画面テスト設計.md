# ホーム画面テスト設計

## 目的

- ホーム画面が画面仕様どおりに「＋ 記録する」「直近1週間の履歴」「履歴を全て見る」を構成することをCIで担保する
- ホーム画面のみ ScreenCard を使わない直接描画へ変更した構成が維持されることを検証する

## 対象スクリプト

- `apps/web/scripts/check-home-screen-spec.mjs`
- 実行コマンド: `pnpm --filter web test:home-screen`

## テストケース

### HS-01 ホーム画面が ScreenCard を使わず直接描画で構成される
- 観点: ホーム専用レイアウトへの移行
- 期待結果: `HomeTabScreen.tsx` に `ScreenCard` が残っておらず、直接描画のラッパーを持つ

### HS-02 ＋ 記録するボタンが画面上部に配置される
- 観点: 主要導線の視認性と文言統一
- 期待結果: `HOME_STRINGS.actions.addRecord` が `＋ 記録する` を保持しつつ履歴セクション見出しより前に描画される

### HS-03 ホーム画面が直近1週間分の履歴取得 hook を利用する
- 観点: 表示件数・取得責務の明確化
- 期待結果: `useRecentHistories(wardrobeId, 7)` を利用して履歴を取得する

### HS-04 履歴セクションが loading / error / empty / list の表示分岐を持つ
- 観点: 非同期取得時のUX維持
- 期待結果: 読み込み中・失敗・空状態・一覧表示の各文言/描画分岐が定義される

### HS-05 ホーム画面の履歴カードが from=home 付きで履歴詳細に遷移する
- 観点: 履歴詳細からホームへ戻る導線維持
- 期待結果: `ROUTES.historyDetail(wardrobeId, item.historyId, "home")` を利用する

### HS-06 ホーム画面下部に履歴一覧導線を配置する
- 観点: 直近履歴から一覧画面への連続導線
- 期待結果: `HOME_STRINGS.actions.viewAllHistories` が履歴セクションの後に描画される

## CI適用

- `.github/workflows/ci.yml` に `Home screen spec test` を追加し、push時に自動検証する


## 追加テスト観点（履歴カード共通デザイン）

### HHCP-01 ホーム画面が履歴一覧と共通の履歴カードを利用する
- 観点: ホーム画面の履歴カードが履歴一覧画面と全く同じデザイン・ロジックを共有できること
- 確認項目:
  - `SharedHistoryCard` を import する
  - `from="home"` を指定して描画する

### HHCP-02 共通履歴カードが日付・右上ラベル・タイトル省略仕様を持つ
- 観点: ホーム画面でも日付書式、補助ラベル、タイトルの情報量制御が履歴一覧と一致すること
- 確認項目:
  - 日付が `formatHistoryDate(item.date)` で表示される
  - 入力方法ラベルが右上の小さめ・薄めスタイルで表示される
  - タイトルが15文字超過時に `...` で省略される

### HHCP-03 共通履歴カードが入力方法別タイトルと詳細導線を切り替える
- 観点: ホーム画面でもテンプレート入力/組み合わせ入力のタイトルルールと詳細遷移先が維持されること
- 確認項目:
  - テンプレート入力はテンプレート名を表示する
  - 組み合わせ入力は服名を `+` 連結で表示する
  - `ROUTES.historyDetail(wardrobeId, item.historyId, from)` を利用する
