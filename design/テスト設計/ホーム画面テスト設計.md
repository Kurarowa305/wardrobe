# ホーム画面テスト設計

## 目的

- ホーム画面が画面仕様どおりに「＋ 記録する」「直近1週間の履歴」「履歴を全て見る」を構成することをCIで担保する
- ホーム画面のみ ScreenCard を使わない直接描画へ変更した構成が維持されることを検証する

## 対象スクリプト

- `apps/web/scripts/check-home-screen-spec.mjs`
- 実行コマンド: `pnpm --filter web test:home-screen`

## テストケース

### HS-01 ホーム画面が ScreenCard を使わず直接描画で構成される
- 観点: ホーム専用レイアウトへの移行
- 期待結果: `HomeTabScreen.tsx` に `ScreenCard` が残っておらず、直接描画のラッパーを持つ

### HS-02 ＋ 記録するボタンが画面上部に配置される
- 観点: 主要導線の視認性と文言統一
- 期待結果: `HOME_STRINGS.actions.addRecord` が `＋ 記録する` を保持しつつ、primary button スタイル（`text-base text-white` / 非太字）で履歴セクション見出しより前に描画される

### HS-03 ホーム画面が直近1週間分の履歴取得 hook を利用する
- 観点: 表示件数・取得責務の明確化
- 期待結果: `useRecentHistories(wardrobeId, HOME_RECENT_HISTORY_LIMIT)` を利用し、`HOME_RECENT_HISTORY_LIMIT = 7` で履歴を取得する

### HS-04 履歴セクションが loading / error / empty / list の表示分岐を持つ
- 観点: 非同期取得時のUX維持
- 期待結果: 読み込み中・失敗・空状態・一覧表示の各文言/描画分岐が定義される

### HS-05 ホーム画面の履歴カードが from=home 付きで履歴詳細に遷移する
- 観点: 履歴詳細からホームへ戻る導線維持
- 期待結果: `ROUTES.historyDetail(wardrobeId, item.historyId, "home")` を利用する

### HS-06 ホーム画面下部に履歴一覧導線を配置する
- 観点: 直近履歴から一覧画面への連続導線
- 期待結果: `HOME_STRINGS.actions.viewAllHistories` が履歴セクションの後に描画される

## CI適用

- `.github/workflows/ci.yml` に `Home screen spec test` を追加し、push時に自動検証する


## 追加テスト観点（履歴カード共通デザイン）

### HHCP-01 ホーム画面が履歴一覧と共通の履歴カードを利用する
- 観点: ホーム画面の履歴カードが履歴一覧画面と全く同じデザイン・ロジックを共有できること
- 確認項目:
  - `SharedHistoryCard` を import する
  - `from="home"` を指定して描画する

### HHCP-02 共通履歴カードが日付表示・入力方法ラベル非表示・タイトル省略仕様を持つ
- 観点: ホーム画面でも日付書式、不要情報の削除、タイトルの情報量制御が履歴一覧と一致すること
- 確認項目:
  - 日付が `formatHistoryDate(item.date)` で表示される
  - 入力方法ラベルを描画しない
  - タイトルが15文字超過時に `...` で省略される

### HHCP-03 共通履歴カードが入力方法別タイトルと詳細導線を切り替える
- 観点: ホーム画面でもテンプレート入力/組み合わせ入力のタイトルルールと詳細遷移先が維持されること
- 確認項目:
  - テンプレート入力はテンプレート名を表示する
  - 組み合わせ入力は服名を `+` 連結で表示する
  - `ROUTES.historyDetail(wardrobeId, item.historyId, from)` を利用する

## 追加テスト観点（直近1週間履歴 hook 利用）

### HRH-01 ホーム画面が useRecentHistories hook を import して利用する
- 観点: 画面コンポーネントが TanStack Query 設計どおりの専用 hook を利用すること
- 確認項目:
  - `useRecentHistories` を import する
  - `const recentHistoriesQuery = useRecentHistories(wardrobeId, HOME_RECENT_HISTORY_LIMIT);` を定義する

### HRH-02 ホーム画面の直近履歴件数が 7 件に固定される
- 観点: ホーム画面の「直近1週間の履歴」と取得件数の対応
- 確認項目:
  - `HOME_RECENT_HISTORY_LIMIT = 7` を定義する

### HRH-03 ホーム画面が useHistoryList / cursor / AutoLoadTrigger に依存しない
- 観点: 専用 hook 利用への置き換えに伴い、旧来のページング制御が除去されること
- 確認項目:
  - `useHistoryList` import が存在しない
  - `cursor` 状態が存在しない
  - `AutoLoadTrigger` を描画しない

### HRH-04 専用テストスクリプトが package.json と CI に登録される
- 観点: 変更内容をローカル/CIの双方で継続検証できること
- 確認項目:
  - `test:home-recent-histories-hook` が `package.json` に存在する
  - `.github/workflows/ci.yml` から `pnpm --filter web test:home-recent-histories-hook` が呼ばれる
