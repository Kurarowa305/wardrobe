# ホーム画面テスト設計

## 目的

- ホーム画面の主要導線が最新要件どおりに構成されていることをCIで継続検証する
- ScreenCard 廃止後も、着た記録導線・直近1週間の履歴表示・履歴一覧遷移が崩れないことを担保する

## 対象スクリプト

- `apps/web/scripts/check-home-screen-spec.mjs`
- 実行コマンド: `pnpm --filter web test:home-screen`

## テストケース

### HS-01 ホーム画面実装が存在する
- 観点: 対象スクリーンファイルの配置
- 期待結果: `src/components/app/screens/HomeTabScreen.tsx` が存在する

### HS-02 着た記録ボタンが画面上部に配置される
- 観点: 最優先アクション導線が履歴より先に描画されるか
- 期待結果: `HOME_STRINGS.actions.addRecord` が履歴セクション見出しより前に出現する

### HS-03 直近1週間の履歴取得 hook を利用する
- 観点: ホーム画面専用の1週間範囲取得ロジックが hooks に実装されているか
- 期待結果:
  - `HomeTabScreen.tsx` で `useRecentWeekHistories` を利用する
  - `src/api/hooks/history.ts` に `from.setDate(from.getDate() - 6)` を含む期間計算がある
  - 取得順は `desc` である

### HS-04 履歴一覧と空状態文言を表示する
- 観点: データ有無に応じた表示切替
- 期待結果:
  - 履歴がある場合は `HomeHistoryCard` の一覧を描画する
  - 履歴がない場合は `HOME_STRINGS.messages.noRecentHistories` を表示する

### HS-05 画面下部に「履歴を全て見る」導線がある
- 観点: 履歴一覧への遷移導線配置
- 期待結果: `HOME_STRINGS.actions.viewAllHistories` が履歴セクションより後に配置され、`ROUTES.histories(wardrobeId)` に遷移する

### HS-06 ScreenCard 依存が除去されている
- 観点: UI基盤刷新の反映
- 期待結果: `HomeTabScreen.tsx` に `ScreenCard` / `ScreenLinkButton` 参照が残っていない

### HS-07 fixture に直近7日分の履歴がある
- 観点: ホーム画面のデモ表示成立性
- 期待結果: `src/mocks/fixtures/history.ts` に `20260321` から `20260315` までの日付 fixture が含まれる

## CI適用

- `.github/workflows/ci.yml` に `Home screen spec test` を追加し、PR時に自動検証する
