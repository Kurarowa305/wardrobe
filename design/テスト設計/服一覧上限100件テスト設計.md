# 服一覧上限100件テスト設計

## 目的
- 服一覧取得から cursor / nextCursor が除去されていることをCIで担保する
- 一覧取得上限100件と登録上限100件の前提が実装に残り続けることを固定化する

## 対象スクリプト
- `apps/web/scripts/check-clothing-list-limit-spec.mjs`
- 実行コマンド: `pnpm --filter web test:clothing-list-limit`

## テストケース
### CLL-01 服一覧 DTO から cursor/nextCursor が削除されている
- 観点: APIスキーマの変更漏れ防止
- 期待結果: `ClothingListParamsDto` に `cursor` がなく、`ClothingListResponseDto` に `nextCursor` がない

### CLL-02 服一覧 query key 正規化から cursor が削除されている
- 観点: キャッシュキーに旧仕様が残らないこと
- 期待結果: `src/api/queryKeys.ts` に cursor 正規化が存在しない

### CLL-03 一覧 hook が nextCursor を扱わない
- 観点: Query hook の責務整理
- 期待結果: `src/api/hooks/clothing.ts` が `nextCursor` を参照しない

### CLL-04 MSW handler が登録上限100件と limit 100 を定義する
- 観点: モック実装で仕様を固定化する
- 期待結果: `DEFAULT_LIST_LIMIT = 100` および `MAX_CLOTHING_COUNT = 100` を持つ

## CI適用
- `.github/workflows/ci.yml` の `Clothing list limit spec test` で自動検証する
