# 削除確認ダイアログ自前実装テスト設計

## 目的

- 削除操作時にブラウザ標準の `window.confirm` ではなく、自前実装の確認ダイアログを利用することをCIで継続検証する
- 服・テンプレート・履歴の各詳細画面で、共通の削除確認UIが利用されることを担保する

## 対象スクリプト

- `apps/web/scripts/check-custom-delete-dialog-spec.mjs`
- 実行コマンド: `pnpm --filter web test:custom-delete-dialog`

## テストケース

### CDD-01 削除確認用の自前ダイアログコンポーネントが存在する
- 観点: 共通UI化とアクセシブルなダイアログ構造
- 期待結果:
  - `src/components/app/dialogs/ConfirmDialog.tsx` に `ConfirmDialog` が定義されている
  - `role="alertdialog"` と `aria-modal="true"` を備える
  - 確認/キャンセル文言を props で切り替えられる

### CDD-02 服詳細画面が ConfirmDialog を利用し window.confirm を使用しない
- 観点: 削除確認導線の置き換え
- 期待結果:
  - `ClothingDetailScreen.tsx` で `ConfirmDialog` を表示する
  - `COMMON_STRINGS.dialogs.confirmDelete.title` と `message` を渡す
  - `window.confirm` を参照しない

### CDD-03 テンプレート詳細画面が ConfirmDialog を利用し window.confirm を使用しない
- 観点: テンプレート削除導線の統一
- 期待結果:
  - `TemplateDetailScreen.tsx` で `ConfirmDialog` を表示する
  - `COMMON_STRINGS.dialogs.confirmDelete.title` と `message` を渡す
  - `window.confirm` を参照しない

### CDD-04 履歴詳細画面が ConfirmDialog を利用し window.confirm を使用しない
- 観点: 履歴削除導線の統一
- 期待結果:
  - `HistoryDetailScreen.tsx` で `ConfirmDialog` を表示する
  - `COMMON_STRINGS.dialogs.confirmDelete.title` と `message` を渡す
  - `window.confirm` を参照しない

## CI適用

- `.github/workflows/ci.yml` に `Custom delete dialog spec test` を追加し、push 時に自動検証する
