# MS4-T06 履歴一覧画面テスト設計

## 目的

- MS4-T06（履歴一覧画面）の完了条件を継続的に検証する
- 履歴一覧の文脈表示、状態表示、サムネイル表示ルールをCIで担保する

## 対象

- `apps/web/src/components/app/screens/HistoriesTabScreen.tsx`
- `apps/web/src/features/history/strings.ts`
- `apps/web/scripts/check-history-list-screen-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### HLS-01 履歴一覧画面が所定ファイルに存在する
- 観点: 一覧画面の実装配置がディレクトリ設計に沿っていること

### HLS-02 useHistoryList と cursor ページングで一覧取得する
- 観点: MS4-T05で作成した Query hook を利用し、一覧の段階的取得に対応していること
- 確認項目:
  - `"use client";` が付与されている
  - `useHistoryList(wardrobeId, { limit, cursor })` を呼ぶ
  - `HISTORY_LIST_PAGE_SIZE = 20` を指定する

### HLS-03 読込中・空状態・読込失敗の表示文言を持つ
- 観点: 履歴一覧の基本状態がユーザーに可視化されること
- 確認項目:
  - `loading`
  - `empty`
  - `error`

### HLS-04 履歴カードが詳細遷移導線と文脈表示を持つ
- 観点: 一覧から履歴詳細へ遷移でき、テンプレート入力/組み合わせ入力の文脈を把握できること
- 確認項目:
  - `ROUTES.historyDetail(wardrobeId, item.historyId, "histories")` を利用する
  - `HISTORY_STRINGS.labels.inputType[item.inputType]` を表示する
  - テンプレート名がない場合に組み合わせ入力向け要約文言を表示する

### HLS-05 履歴カードがサムネ表示を持ち、画像なし/削除済みに対応する
- 観点: 履歴に含まれる服の概要を一覧で把握できること
- 確認項目:
  - `resolveImageUrl` で画像URLを解決する
  - 画像なし時に `no image` を表示する
  - 削除済み服に `削除済み` オーバーレイを表示する

### HLS-06 サムネイル最大4件と超過分 `+x` を表示する
- 観点: テンプレート一覧と同等のサムネイル表示ルールで情報量を制御できること
- 確認項目:
  - `HISTORY_THUMBNAIL_LIMIT = 4`
  - `slice(0, HISTORY_THUMBNAIL_LIMIT)` を使う
  - 超過件数を `+{hiddenCount}` で表示する

### HLS-07 `nextCursor` がある場合に「さらに読み込む」で追加取得する
- 観点: 履歴一覧APIのページングに応じて継続読み込みできること
- 確認項目:
  - `setNextCursor(data.nextCursor)` を保持する
  - ボタン押下で `setCursor(nextCursor)` を呼ぶ
  - `HISTORY_STRINGS.list.actions.loadMore` を表示する

### HLS-08 一覧画面向け文言が history strings に定義される
- 観点: 一覧固有の文言を画面から分離し、他画面と同様に集中管理できること
- 確認項目:
  - `loadMore`
  - `loading`
  - `empty`
  - `error`
  - `combinationSummary`
  - `deleted`

## 追加テスト観点（履歴カード表示調整）

### HLCP-01 日付が `YYYY/MM/DD` 形式で表示される
- 観点: 履歴カードの日付表示が詳細画面と同じ視認しやすい形式で統一されること
- 確認項目:
  - `formatHistoryDate(item.date)` を利用する
  - `src/features/history/date.ts` に `YYYY/MM/DD` 変換関数が存在する

### HLCP-02 入力方法ラベルがカード右上に控えめなスタイルで表示される
- 観点: テンプレート入力/組み合わせ入力の文脈を補助情報として目立ちすぎず表示できること
- 確認項目:
  - カード先頭行が `justify-between` で左右配置される
  - 入力方法ラベルに小さめ文字サイズと薄めの文字色を指定する

### HLCP-03 タイトルが入力方法に応じて切り替わる
- 観点: カードの主見出しが記録方法に応じた内容になること
- 確認項目:
  - テンプレート入力ではテンプレート名を表示する
  - 組み合わせ入力では服名を `+` で連結して表示する
  - フォールバックとして組み合わせ要約文言を利用できる

### HLCP-04 タイトルが15文字超過時に `...` で省略される
- 観点: 長いテンプレート名や服名連結でもカード高さを過度に増やさないこと
- 確認項目:
  - `HISTORY_CARD_TITLE_MAX_LENGTH = 15` を定義する
  - 15文字を超えた場合に `...` を付ける
