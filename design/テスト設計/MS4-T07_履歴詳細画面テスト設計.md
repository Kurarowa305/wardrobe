# MS4-T07 履歴詳細画面テスト設計

## 目的
- 履歴詳細画面が History API の詳細取得結果を用いて、表示専用のUIを設計どおりに構成していることを静的検証する。
- 戻り先文脈付きルーティングを維持したまま、履歴詳細ページが query からIDを解決できることを確認する。

## 対象
- `apps/web/src/components/app/screens/HistoryDetailScreen.tsx`
- `apps/web/src/app/histories/detail/page.tsx`
- `apps/web/src/features/history/strings.ts`
- `apps/web/scripts/check-history-detail-screen-spec.mjs`

## テスト観点
1. `useHistory(wardrobeId, historyId)` を利用した詳細取得と、ロード中・エラー時の表示分岐を持つ。
2. 履歴詳細が日付、入力方法、テンプレート入力時の着用テンプレート、着用服一覧を表示し、「最後に着た日」行を含まない。
3. 着用服一覧は服詳細へのリンクを持たず、画像URL解決・no image・削除済み表示に対応し、「最後に着た日」行を表示しない。
4. ヘッダーメニューに削除項目のみが存在し、編集項目が含まれない。
5. 履歴詳細ページが `wardrobeId/historyId` を query から解決し、`HistoryDetailScreen` に渡す。
6. 履歴詳細向け文言が `history/strings.ts` に定義され、未使用の「最後に着た日」関連文言を含まない。

## テストケース
- **HDS-01**: 履歴詳細画面ファイルが存在する。
- **HDS-02**: `useHistory` を利用し、`isPending` / `isError` / エラー表示分岐を備える。
- **HDS-03**: 日付・入力方法・着用テンプレート（テンプレート入力時のみ）・着用服一覧・着用回数を表示し、最後に着た日行を表示しない。
- **HDS-04**: 着用服一覧が非リンク表示で、画像・プレースホルダ・削除済み表示に対応する。
- **HDS-05**: ヘッダーメニューに削除のみがあり、編集が含まれない。
- **HDS-06**: 詳細ページが query から `wardrobeId/historyId` を解決し、screen に渡す。
- **HDS-07**: 履歴詳細向け文言が strings に定義され、最後に着た日関連文言を含まない。

## 実行コマンド
- `pnpm --filter web test:history-detail-screen`
