# 一覧画面余白調整テスト設計

## 目的

* 履歴、テンプレート、服の一覧画面で「さらに読み込む」ボタンの上余白が確保されていることを確認する。
* テンプレート一覧画面と服一覧画面で追加ボタンの下余白が確保されていることを確認する。
* 既存のアクセントカラー検証テストと競合しないよう、ボタン本体のスタイルではなく余白コンテナで調整されていることを確認する。

## テスト対象

* `apps/web/src/components/app/screens/HistoriesTabScreen.tsx`
* `apps/web/src/components/app/screens/TemplatesTabScreen.tsx`
* `apps/web/src/components/app/screens/ClothingsTabScreen.tsx`
* `apps/web/scripts/check-list-screen-spacing-spec.mjs`

## テスト観点

1. 履歴一覧画面の「さらに読み込む」ボタン直前に `mt-4` を持つ余白コンテナが設定されている。
2. テンプレート一覧画面の「＋ テンプレートを追加」ボタンを `mb-4` の余白コンテナで囲っている。
3. テンプレート一覧画面の「さらに読み込む」ボタン直前に `mt-4` を持つ余白コンテナが設定されている。
4. 服一覧画面の「＋ 服を追加」ボタンを `mb-4` の余白コンテナで囲っている。
5. 服一覧画面の「さらに読み込む」ボタン直前に `mt-4` を持つ余白コンテナが設定されている。

## 実行方法

```bash
pnpm --filter web test:list-screen-spacing
pnpm --filter web test:accent-color
```

## CI適用方針

* GitHub Actions の `CI` workflow で `pnpm --filter web test:list-screen-spacing` と既存の `pnpm --filter web test:accent-color` が両立して通過する状態を維持する。
