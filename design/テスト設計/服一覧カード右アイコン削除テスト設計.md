# 服一覧カード右アイコン削除テスト設計

## 目的
- 服一覧カード右側の補助アイコンが削除され、カード本文が画像と服名に集中した表示になることを継続的に検証する。
- テンプレート作成や記録画面で使う選択式カードのチェックマークUIを維持し、共通コンポーネント改修の副作用を防ぐ。

## 対象
- `apps/web/src/components/app/screens/ClothingGenreSection.tsx`
- `apps/web/scripts/check-clothing-list-right-icon-removal-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### CLRI-01 服一覧カード実装ファイルが存在する
- 観点: 監視対象ファイルの欠落防止。
- 検証内容:
  - `src/components/app/screens/ClothingGenreSection.tsx` が存在する。

### CLRI-02 通常の服一覧カードが右アイコンなしの2カラムレイアウトを使う
- 観点: 一覧カードの見た目が画像＋服名に簡素化されていること。
- 検証内容:
  - 非選択時は `grid-cols-[56px_minmax(0,1fr)]` を使う。
  - 選択時のみ `grid-cols-[56px_minmax(0,1fr)_40px]` を使う。

### CLRI-03 通常の服一覧カードは右端アイコンを描画しない
- 観点: ユーザー要望どおり一覧カード右側のアイコンが除去されていること。
- 検証内容:
  - 非選択カード向けの `ClothingGenreIcon` 描画が存在しない。

### CLRI-04 選択式カードのチェックマークUIは維持する
- 観点: 共通コンポーネントを利用する他画面の選択操作を壊さないこと。
- 検証内容:
  - `selectable` 分岐が残っている。
  - チェックマーク `✓` の描画が残っている。
  - 選択時3カラムレイアウトが維持される。

## 実行方法
- `pnpm --filter web test:clothing-list-right-icon-removal`

## CI適用方針
- GitHub Actions `CI` ワークフローに `Clothing list right icon removal spec test` を追加し、既存のWeb向け設計同期チェックと同列で毎回実行する。
