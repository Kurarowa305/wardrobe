# ホームヘッダーワードローブ名表示テスト設計

## 目的

- ホーム画面のヘッダー中央が固定文言ではなく、API-02 で取得したワードローブ名を表示することを継続検証する。
- 取得中・取得失敗時はヘッダー中央を空欄にする方針が崩れないことを静的テストで担保する。

## 対象スクリプト

- `apps/web/scripts/check-home-header-wardrobe-title-spec.mjs`
- 実行コマンド: `pnpm --filter web test:home-header-wardrobe-title`

## テストケース

### HWT-01 useWardrobeDetail が queryKeys.wardrobe.detail と getWardrobe を利用する
- 観点: ホーム専用のワードローブ詳細取得責務を hook に閉じ込める
- 期待結果:
  - `useWardrobeDetail(wardrobeId)` が存在する
  - `queryKeys.wardrobe.detail(wardrobeId)` を利用する
  - `getWardrobe(wardrobeId)` を `queryFn` に使う
  - `enabled: wardrobeId.length > 0` を持つ

### HWT-02 getWardrobe が GET /wardrobes/{wardrobeId} を呼び name を検証する
- 観点: API-02 の contract を frontend 側で崩さない
- 期待結果:
  - `/wardrobes/${wardrobeId}` を呼ぶ
  - `WardrobeDetailResponseDto` を返す
  - `name: string` を検証する
  - 不正レスポンス時に `INVALID_RESPONSE` を投げる

### HWT-03 HomeTabScreen が useWardrobeDetail を呼び出す
- 観点: ホーム画面が固定文言ではなく API 結果を参照する
- 期待結果:
  - `HomeTabScreen.tsx` が `useWardrobeDetail` を import する
  - `const wardrobeQuery = useWardrobeDetail(wardrobeId);` を持つ

### HWT-04 ホームの AppLayout タイトルが成功時のみワードローブ名で、それ以外は空文字になる
- 観点: 表示方針「成功時のみ表示」を維持する
- 期待結果:
  - `title: wardrobeQuery.data?.name ?? ""` を使う
  - 旧来の固定タイトル代入が残っていない

### HWT-05 HOME_STRINGS.titlePlaceholder が残っていない
- 観点: ホームタイトルの固定プレースホルダー依存を除去する
- 期待結果:
  - `src/features/home/strings.ts` に `titlePlaceholder` が存在しない

### HWT-06 package script と CI に新規テストが接続されている
- 観点: ローカルと CI の双方で退行を検知できる
- 期待結果:
  - `apps/web/package.json` に `test:home-header-wardrobe-title` がある
  - `.github/workflows/ci.yml` で `pnpm --filter web test:home-header-wardrobe-title` が実行される

## CI適用

- `.github/workflows/ci.yml` に `Home header wardrobe title spec test` を追加し、push 時に自動検証する

- HWT-03 `HomeTabScreen` が `useWardrobeDetail` を利用
- HWT-04 タイトルが成功時のみワードローブ名、それ以外は空文字
- HWT-05 `HOME_STRINGS.titlePlaceholder` の除去
