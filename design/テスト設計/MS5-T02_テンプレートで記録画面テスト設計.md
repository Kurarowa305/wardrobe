# MS5-T02 テンプレートで記録画面テスト設計

## 1. 対象

- `apps/web/src/components/app/screens/RecordByTemplateScreen.tsx`
- `apps/web/src/features/record/strings.ts`
- `apps/web/scripts/check-record-by-template-screen-spec.mjs`
- `.github/workflows/ci.yml`

## 2. テスト観点

1. テンプレートで記録画面が client component として実装されていること。
2. `useTemplateList` を `limit` / `cursor` 付きで利用し、取得済みページを蓄積できること。
3. 日付入力（必須）とテンプレート単一選択 UI が揃っていること。
4. 記録ボタン押下時に `useCreateHistoryMutation` で `date` と `templateId` を送信し、成功後にホームへ戻ること。
5. 読込中 / 空状態 / 読込失敗 / 必須エラー / 送信失敗の表示文言が実装されていること。
6. 「さらに読み込む」と「キャンセル」の導線が存在すること。
7. 必要な文言定義が `record/strings.ts` に追加され、CI からテストスクリプトが実行されること。

## 3. テストケース

| ID | 観点 | 確認内容 |
| --- | --- | --- |
| RBTS-01 | 画面存在 | `RecordByTemplateScreen` が client component として定義される |
| RBTS-02 | 一覧取得 | `useTemplateList` が `limit=20` / `cursor` で呼ばれ、ページ蓄積処理を持つ |
| RBTS-03 | UI | `type="date"` と `type="radio"` による単一選択 UI を持つ |
| RBTS-04 | 送信 | `useCreateHistoryMutation` へ `date` / `templateId` を渡し、成功後 `ROUTES.home` へ遷移する |
| RBTS-05 | 状態表示 | 読込・空・エラー・必須・送信失敗の各文言が参照される |
| RBTS-06 | 導線 | 追加読込ボタンとキャンセルボタンがある |
| RBTS-07 | 文言/CI | `record/strings.ts` と CI に必要な設定が存在する |

## 4. 実行方法

```bash
pnpm --filter web test:record-by-template-screen
```

## 5. CI適用

- GitHub Actions の `CI` workflow に `Record by template screen spec test` を追加し、プッシュ時に自動検証する。
