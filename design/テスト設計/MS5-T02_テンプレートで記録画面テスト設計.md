# MS5-T02 テンプレートで記録画面テスト設計

## 目的

- MS5-T02（記録：テンプレで記録）の完了条件を継続的に検証する
- テンプレート選択から履歴作成までの最短導線が崩れないことをCIで担保する

## 対象

- `apps/web/src/components/app/screens/RecordByTemplateScreen.tsx`
- `apps/web/src/features/record/strings.ts`
- `apps/web/src/app/wardrobes/[wardrobeId]/(stack)/record/template/page.tsx`
- `apps/web/scripts/check-record-by-template-screen-spec.mjs`
- `.github/workflows/ci.yml`

## テスト観点

### RBTS-01 テンプレート記録画面が所定ファイルに存在する
- 観点: 記録フローのテンプレート選択画面が設計どおりの場所にあること

### RBTS-02 client component として template/history hooks を接続する
- 観点: 画面がクライアント側で一覧取得と履歴作成を扱えること
- 確認項目:
  - `useTemplateList(wardrobeId, { limit, cursor })`
  - `useCreateHistoryMutation(wardrobeId)`

### RBTS-03 AppLayout のタイトル・戻り導線を定義する
- 観点: 画面がスタック画面としてヘッダーと方法選択画面への戻り導線を持つこと
- 確認項目:
  - `title={RECORD_STRINGS.byTemplate.title}`
  - `backHref={ROUTES.recordMethod(wardrobeId)}`

### RBTS-04 日付入力とテンプレート単一選択 UI を表示する
- 観点: 仕様書どおり「日付」「テンプレート単一選択」「記録」から構成されること
- 確認項目:
  - `type="date"`
  - `type="radio"`
  - `name="templateId"`

### RBTS-05 必須バリデーションで未入力送信を防ぐ
- 観点: 日付未入力・テンプレート未選択のまま記録できないこと
- 確認項目:
  - `dateRequired`
  - `templateRequired`
  - `disabled={isDateEmpty || isTemplateEmpty || isPending}`

### RBTS-06 記録実行時に createHistoryMutation へ date/templateId を渡し、成功後にホームへ遷移する
- 観点: テンプレート記録が履歴作成 API に接続され、完了後にホームへ戻ること
- 確認項目:
  - `await createHistoryMutation.mutateAsync({ date, templateId })`
  - `router.push(ROUTES.home(wardrobeId))`

### RBTS-07 テンプレート一覧の状態表示と追加読み込みを持つ
- 観点: loading/error/empty とページング UI が定義されていること
- 確認項目:
  - `loading`
  - `loadError`
  - `empty`
  - `loadMore`
  - `nextCursor`

### RBTS-08 ページファイルが RecordByTemplateScreen を描画する
- 観点: ルートから画面コンポーネントへ正しく接続されていること

### RBTS-09 画面固有文言が record strings に集約される
- 観点: バリデーション・状態表示・送信状態が文言定義に集中管理されること
- 確認項目:
  - `loading`
  - `loadError`
  - `empty`
  - `dateRequired`
  - `templateRequired`
  - `submitting`
  - `submitError`

## テスト実装

- スクリプト: `pnpm --filter web test:record-by-template-screen`
- 実体: `apps/web/scripts/check-record-by-template-screen-spec.mjs`
- CI適用:
  - `.github/workflows/ci.yml` に `Record by template screen spec test` を追加し、push時に自動検証する

- RBTS-04 日付入力 + テンプレート単一選択 UI
- RBTS-05 必須バリデーションと送信抑止
- RBTS-06 createHistoryMutation 実行 + ホーム遷移

- RBTS-08 page.tsx からの画面描画
