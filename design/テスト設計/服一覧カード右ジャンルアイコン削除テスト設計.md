# 服一覧カード右ジャンルアイコン削除テスト設計

## 目的

- 服タブ一覧カード右側に表示されていたジャンルアイコンが削除されていることを継続的に検証する。
- ジャンル別セクション見出しのアイコンや、選択UIで使う右端列は維持されることをあわせて担保する。
- 変更内容をCIに組み込み、一覧カード表示の退行を早期に検知できる状態にする。

## 対象スクリプト

- `apps/web/scripts/check-clothing-list-card-genre-icon-removal-spec.mjs`
- 実行コマンド: `pnpm --filter web test:clothing-list-card-genre-icon-removal`

## テストケース

### CLGI-01 服一覧カード共通実装が存在する
- 観点: 対象コンポーネントの配置確認
- 期待結果: `apps/web/src/components/app/screens/ClothingGenreSection.tsx` が存在する

### CLGI-02 非選択カードが2カラム構成になっている
- 観点: 右端アイコン削除に伴うレイアウト更新
- 期待結果:
  - `selectable` が `false` の場合は `grid-cols-[56px_minmax(0,1fr)]` を利用する
  - `selectable` が `true` の場合のみ `grid-cols-[56px_minmax(0,1fr)_40px]` を利用する

### CLGI-03 非選択カードに右端ジャンルアイコンが残っていない
- 観点: 服タブ一覧カードの右端装飾削除
- 期待結果:
  - 非選択カード描画で `ClothingGenreIcon` の右端表示が存在しない
  - 選択UI用の右端領域はチェックマーク表示にのみ使われる

### CLGI-04 セクション見出しのジャンルアイコンは維持される
- 観点: ジャンル別セクション要件の維持
- 期待結果:
  - セクション見出しでは引き続き `ClothingGenreIcon` が表示される
  - ジャンル見出しと折りたたみ導線の仕様に影響しない

## CI適用

- `.github/workflows/ci.yml` に `Clothing list card genre icon removal spec test` を追加し、push / PR 相当のCIで自動検証する。
