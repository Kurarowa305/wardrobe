# 開発ツール利用ガイド

## 目的

- 本リポジトリで日常的に使う開発ツールの実行方法を一箇所に集約する
- ローカル検証と CI 実行内容を揃え、差分のない確認手順を提供する

## 1. パッケージ管理（pnpm）

- 前提バージョン: `pnpm@10.27.0`（`packageManager` に定義）
- インストール:

```bash
pnpm install
```

- lockfile 整合性確認（CIと同一）:

```bash
pnpm install --frozen-lockfile
```

- ワークスペース指定実行:

```bash
pnpm --filter web <script>
pnpm --filter api <script>
```

## 2. アプリ起動

### 2.1 Web（Next.js）

```bash
pnpm --filter web dev
```

- 本番ビルド:

```bash
pnpm --filter web build
pnpm --filter web start
```

### 2.2 API（Node + TypeScript）

```bash
pnpm --filter api dev
```

- `apps/api/src/index.ts` を `tsx watch` で監視起動する

## 3. 型チェック・テスト（CI対応）

### 3.1 基本チェック

```bash
pnpm -r --if-present run lint
pnpm -r --if-present run test
pnpm --filter api exec tsc --noEmit
pnpm --filter api exec tsc --outDir dist
pnpm --filter web exec tsc --noEmit
```

## 4. MSW（モックAPI）運用

- 開発時の `GET */health` は MSW でモックできる
- クエリパラメータでシナリオを再現可能

### 4.1 遅延シナリオ

- `delay=<ミリ秒>` を付与してレスポンス遅延を発生
- 例: `/health?delay=1500`

### 4.2 強制エラーシナリオ

- `forceError` で 404 / 500 を強制
- 主な値:
  - `forceError=500`
  - `forceError=true`
  - `forceError=404`
  - `forceError=notfound`
- 例: `/health?delay=800&forceError=404`

### 4.3 ページ上で `fetch` を実行して確認する手順

1. 開発サーバーを起動する

```bash
pnpm --filter web dev
```

2. ブラウザで `http://localhost:3000/wardrobes/new` を開く
3. DevTools を開く（Chrome: `F12` または `Ctrl+Shift+I` / `Cmd+Option+I`）
4. `Console` タブで以下を実行する

```js
await fetch("/health").then(async (r) => ({ status: r.status, body: await r.json() }));
await fetch("/health?delay=1500").then(async (r) => ({ status: r.status, body: await r.json() }));
await fetch("/health?forceError=500").then(async (r) => ({ status: r.status, body: await r.json() }));
await fetch("/health?delay=800&forceError=404").then(async (r) => ({ status: r.status, body: await r.json() }));
```

5. `Network` タブで各リクエストの `Status` と `Response` を確認する

- 補足:
  - `/health` をアドレスバーで直接開くとページ遷移（navigate）扱いになり、MSWではなくNext.jsのルーティング解決になる
  - 相対パス `health` は現在URL配下（例: `/wardrobes/new/health`）になるため、必ず先頭スラッシュ付きの `/health` を使う

## 5. DynamoDB Local（Docker）

- 定義ファイル: `docker/dynamodb/docker-compose.yml`
- 起動:

```bash
cd docker/dynamodb
docker compose up -d
```

- 状態確認:

```bash
docker compose ps
docker compose logs -f
```

- 停止:

```bash
docker compose down
```

- 接続先: `http://localhost:8000`

## 6. Terraform（インフラ）

### 6.1 bootstrap（tfstate バケット作成）

```bash
cd infra/terraform/bootstrap
terraform init
terraform plan \
  -var='env=dev' \
  -var='region=ap-northeast-1' \
  -var='name_prefix=wardrobe-dev-'
terraform apply \
  -var='env=dev' \
  -var='region=ap-northeast-1' \
  -var='name_prefix=wardrobe-dev-'
```

### 6.2 app（本体インフラ）

- 先に Lambda zip を作成:

```bash
pnpm --filter api exec tsc --outDir ../../infra/terraform/app/build --rootDir src
cd infra/terraform/app
zip -j build/lambda.zip build/index.js
```

- Plan / Apply:

```bash
terraform init
terraform workspace select dev || terraform workspace new dev
terraform plan -var-file=../env/dev.tfvars
terraform apply -var-file=../env/dev.tfvars
```
