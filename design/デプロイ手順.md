# デプロイ手順

## 1. 目的

本書は、`wardrobe` プロジェクトを手動で AWS `dev` 環境へデプロイする手順をまとめる。

## 2. 前提

- 実行ディレクトリはリポジトリルート: `/home/oyadomaritaiki/wardrobe`
- AWS 認証が済んでいること（`aws sts get-caller-identity` が成功すること）
- `pnpm install --frozen-lockfile` が成功すること
- 対象リージョン: `ap-northeast-1`
- 対象 Terraform 変数ファイル: `infra/terraform/env/dev.tfvars`

初期確認:

```bash
cd /home/oyadomaritaiki/wardrobe
export AWS_REGION=ap-northeast-1
# 必要なら export AWS_PROFILE=<your-profile>
aws sts get-caller-identity
pnpm install --frozen-lockfile
```

## 3. 手動デプロイ手順

### 3.1 初回のみ: bootstrap（tfstate バケット作成）

`tfstate` バケットが未作成の場合のみ実行する。

```bash
cd infra/terraform/bootstrap
terraform init
terraform plan \
  -var='env=dev' \
  -var='region=ap-northeast-1' \
  -var='name_prefix=wardrobe-dev-'
terraform apply \
  -var='env=dev' \
  -var='region=ap-northeast-1' \
  -var='name_prefix=wardrobe-dev-'
cd ../../..
```

### 3.2 Lambda パッケージ作成

```bash
bash infra/terraform/app/scripts/package-lambda.sh
```

### 3.3 Terraform(app) 反映

```bash
terraform init
terraform workspace select dev || terraform workspace new dev
terraform plan -var-file=../env/dev.tfvars
terraform apply -var-file=../env/dev.tfvars

WEB_BUCKET_NAME=$(terraform output -raw web_bucket_name)
WEB_DIST_ID=$(terraform output -raw web_cdn_distribution_id)
WEB_CDN_DOMAIN=$(terraform output -raw web_cdn_domain)
API_ENDPOINT=$(terraform output -raw api_endpoint)
WEB_TOP_URL="https://${WEB_CDN_DOMAIN}/"
cd ../../..
```

### 3.4 Web 配信反映（S3 sync + CloudFront invalidation）

```bash
export NEXT_PUBLIC_API_BASE_URL=https://izkkveuro2.execute-api.ap-northeast-1.amazonaws.com
export NEXT_PUBLIC_IMAGE_PUBLIC_BASE_URL=https://d3es6nwimv8wfx.cloudfront.net
export NEXT_PUBLIC_ENABLE_MSW=false
pnpm --filter web build

if [ -d apps/web/out ]; then
  aws s3 sync apps/web/out s3://${WEB_BUCKET_NAME}
  aws cloudfront create-invalidation --distribution-id "${WEB_DIST_ID}" --paths "/*"
else
  echo "apps/web/out が存在しないため static 配信は実施しません（SSRモード）。"
fi

echo "Top page URL: ${WEB_TOP_URL}"
echo "API endpoint: ${API_ENDPOINT}"
```

## 4. デプロイ後確認

- `WEB_TOP_URL` にアクセスし、トップページが表示されることを確認
- `API_ENDPOINT` への疎通確認
- 必要に応じて CloudWatch Logs / API Gateway / CloudFront のメトリクスを確認

## 5. 運用上の注意

- プロジェクト方針では `terraform apply` は CI 実行が原則。
- 手動 `apply` は障害対応や検証など、例外時のみ実行する。
- Terraform state は S3 backend を使用し、同時 `apply` を避ける。
