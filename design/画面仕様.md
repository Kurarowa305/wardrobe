# ワードローブアプリ

## 画面仕様書

---

## 0. 本資料の位置づけ

本資料は、ワードローブ管理Webアプリ（スマートフォン前提）における
**画面構成・表示要素・操作仕様・画面遷移**を定義する。

* デザイン（色・余白・フォント等）は本資料の対象外とする
* 画像はあくまでイメージであり、仕様の正とはしない
* 今後の画面追加・機能拡張を前提とする

---

## 1. 共通仕様

### 1.1 対象デバイス

* スマートフォン（縦持ち）
* タッチ操作前提

---

### 1.2 ヘッダー仕様（全画面共通）

| 画面種別                  | 左  | 中央      | 右       |
| --------------------- | -- | ------- | ------- |
| ホーム                   | なし | ワードローブ名 | なし      |
| タブ一覧（履歴 / テンプレート / 服） | なし | 画面名称    | なし      |
| 詳細 / 追加 / 編集 / 記録     | 戻る | 画面名称    | 必要に応じて︙ |

* 画面名称は日本語表記を基本とする

  * 例：履歴、履歴詳細、テンプレート、服、記録、服の編集 など

---

### 1.3 タブバー

* 表示対象

  * ホーム
  * 履歴一覧
  * テンプレート一覧
  * 服一覧

* 非表示

  * 詳細 / 追加 / 編集 / 記録 / 履歴詳細

---

### 1.4 一覧カード共通仕様

* カード全体がタップ領域

* サムネイル表示ルール

  * 画像あり：画像表示
  * 画像なし：`no image`
  * 削除済み：`削除済み` オーバーレイ

* サムネ表示上限

  * 最大4件
  * 超過分は `+x` 表示

---

### 1.5 確認モーダル

#### 削除確認

* タイトル：削除しますか？
* 本文：この操作は取り消せません
* ボタン：キャンセル / 削除

#### 入力破棄確認

* タイトル：入力を破棄しますか？
* 本文：入力途中の内容は保存されません
* ボタン：キャンセル / 破棄

## 3. 各画面仕様

---

## 3.1 ワードローブ作成

### UI

* ワードローブ名（必須）
* 作成ボタン

### 挙動

* 作成成功後、ホームへ遷移

---

## 3.2 ホーム

### ヘッダー

* 中央：ワードローブ名

### UI構成

1. `＋ 着た記録` ボタン
2. 直近1週間の履歴
3. `履歴を全て見る` 導線

### 履歴カード

* 日付
* 入力種別ラベル

  * テンプレート入力 / 組み合わせ入力
* タイトル
* サムネ（最大4件 + `+x`）

---

## 3.3 履歴一覧

### UI

* 履歴カードの縦リスト
* 下スクロールで追加ロード（週単位）

---

## 3.4 履歴詳細

### ヘッダー

* 中央：履歴詳細
* 右：︙（編集 / 削除）

### UI構成

1. 日付表示
2. 着用服一覧（縦リスト）

### 着用服表示

* サムネイル
* 服名
* 記録時の選択順を保持

### 操作仕様

* **服カードはタップ不可**
* **履歴詳細画面から服の詳細画面へは遷移しない**

### 削除

* 物理削除
* 削除後は履歴一覧へ戻る

---

## 3.5 テンプレート一覧

### UI

* `＋ テンプレートを追加`
* テンプレートカード一覧

---

## 3.6 テンプレート追加 / 編集（共通）

### UI

* テンプレート名（必須）
* 構成する服を選択（複数）
* 送信ボタン

  * 追加：追加
  * 編集：保存

### 仕様

* 編集画面では、現在の入力内容を初期値として表示する

---

## 3.7 テンプレート詳細

### UI

* テンプレート名
* 構成服一覧
* ︙（編集 / 削除）

---

## 3.8 服一覧

### UI

* `＋ 服を追加`
* 服カード一覧

---

## 3.9 服追加 / 編集（共通）

### UI

* 画像（任意）
* 服の名前（必須）
* 送信ボタン

  * 追加：追加
  * 編集：保存

### 仕様

* 編集画面では、現在の入力内容を初期値として表示する

---

## 3.10 服詳細

### UI

* 画像
* 服名
* ︙（編集 / 削除）

---

## 3.11 記録（方法選択）

### UI

* テンプレートで記録
* 服の組み合わせで記録

---

## 3.12 記録（テンプレートで記録）

### UI

1. 日付入力（必須）
2. テンプレート選択（単一）
3. 記録ボタン

---

## 3.13 記録（服の組み合わせで記録）

### UI

1. 日付入力（必須）
2. 服選択（複数）
3. 記録ボタン

---

## 4. 削除ルールまとめ

| 対象     | 削除方式 |
| ------ | ---- |
| 服      | 論理削除 |
| テンプレート | 論理削除 |
| 履歴     | 物理削除 |

* 削除済み服は

  * テンプレート / 履歴 / 履歴詳細では表示
  * 服一覧では非表示

---

## 5. 補足設計方針

* 編集画面は「追加画面の流用」を基本とする
* 差分は

  * 初期値
  * ボタン文言
  * 実行API
    のみとする
* 一覧・詳細は「確認」に専念し、編集は明示的に遷移させる
